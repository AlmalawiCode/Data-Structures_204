Introduction to Algorithm Analysis

Why Algorithm Analysis?

Algorithm analysis helps us understand:

  • Time Complexity: How much time an algorithm takes
  • Space Complexity: How much memory an algorithm uses
  • Efficiency: How well an algorithm performs as input size grows

Common Complexity Classes

Function Name Example
O(1) Constant Array access
O(log n) Logarithmic Binary search
O(n) Linear Linear search
O(n log n) Linearithmic Merge sort
O(n²) Quadratic Nested loops
O(n³) Cubic Triple nested loops
O(2ⁿ) Exponential Recursive fibonacci
O(n!) Factorial Permutations

Step 1: Converting Java Code to Polynomial Equations

Basic Rules

  1. Simple Statement: Each statement = 1 operation
  2. Loop: Multiply iterations by operations inside
  3. Nested Loops: Multiply all loop iterations together
  4. Sequential Code: Add operations together

Example 1: Single Loop

for (int i = 0; i < n; i++) {
    sum = sum + i;  // 1 operation
}

Analysis:

Loop runs: n times

Operations per iteration: 1

Total operations: T(n) = n × 1 = n

f(n) = n

Example 2: Nested Loops

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        matrix[i][j] = 0;  // 1 operation
    }
}

Analysis:

Outer loop runs: n times

Inner loop runs: n times (for each outer iteration)

Operations per inner iteration: 1

Total operations: T(n) = n × n × 1 = n²

f(n) = n²

Example 3: Sequential Loops

for (int i = 0; i < n; i++) {
    sum += i;  // 1 operation
}

for (int j = 0; j < n*n; j++) {
    delta += 1;  // 1 operation
}

Analysis:

First loop: n operations

Second loop: n² operations

Total operations (add them): T(n) = n + n²

f(n) = n² + n

Example 4: Logarithmic Loop

while (n > 0) {
    System.out.println(n);
    n = n / 2;
}

Analysis:

Initial value: n

After 1st iteration: n/2

After 2nd iteration: n/4

After 3rd iteration: n/8

After kth iteration: n/2k

Loop ends when: n/2k = 1, so k = log₂(n)

f(n) = log₂(n)

Example 5: Complex Nested Loop

for (int i = 0; i < n; i++) {
    for (int j = 0; j < i; j++) {
        sum += i * j;  // 1 operation
    }
}

Analysis:

Outer loop: runs n times

Inner loop: runs 0, 1, 2, ..., n-1 times

Total: 0 + 1 + 2 + ... + (n-1) = n(n-1)/2

Expanding: (n² - n)/2 = n²/2 - n/2

f(n) = (1/2)n² - (1/2)n

Step 2: Finding Big O Notation from Polynomials

Rules for Converting to Big O

  1. Keep only the highest order term (term with largest exponent)
  2. Drop all coefficients (constants multiplying the term)
  3. Drop all lower order terms

Why These Rules?

As n grows large, the highest order term dominates. For example:

If f(n) = 4n² + 3n + 10

  • When n = 10: 4(100) + 3(10) + 10 = 440
  • When n = 1000: 4(1,000,000) + 3(1000) + 10 = 4,003,010
  • The n² term (4,000,000) dominates!

Example 1

f(n) = 4n² + 3n + 10

Step 1: Identify highest order term → 4n²

Step 2: Drop coefficient → n²

Step 3: Drop lower terms → (already done)

Big O = O(n²)

Example 2

f(n) = 76,756,234n² + 427,913n + 7

Step 1: Highest order term → 76,756,234n²

Step 2: Drop coefficient → n²

Big O = O(n²)

Example 3

f(n) = 74n⁸ - 62n⁵ - 71,562n³ + 3n² - 5

Step 1: Highest order term → 74n⁸

Step 2: Drop coefficient → n⁸

Big O = O(n⁸)

Example 4

f(n) = 42n⁴(12n⁶ - 73n² + 11)

Step 1: Expand → 42n⁴ × 12n⁶ = 504n¹⁰ (highest term)

Step 2: Drop coefficient → n¹⁰

Big O = O(n¹⁰)

Example 5

f(n) = 75n log(n) - 415

Step 1: Highest order term → 75n log(n)

Step 2: Drop coefficient → n log(n)

Big O = O(n log n)

Example 6: Constant Time

f(n) = 5

Analysis: No n term, constant operations

Big O = O(1)

Step 3: Calculating Algorithm Execution Time

Formula

If T(n) = O(g(n)), then:

T(n) = c × g(n)

Where c is a constant that depends on the implementation and hardware.

Process

  1. Use known time for one input size to find constant c
  2. Apply c to predict time for different input sizes

Example 1: O(n) Algorithm

Given:

  • Algorithm runs in O(n) time
  • For n = 10, time = 2 ms

Find: Time when n = 500

Solution:

Step 1: T(n) = c × n

Step 2: Find c using known values:

2 = c × 10

c = 2/10 = 1/5

Step 3: Calculate T(500):

T(500) = (1/5) × 500 = 100 ms

Answer: 100 milliseconds

Example 2: O(n²) Algorithm

Given:

  • Algorithm runs in O(n²) time
  • For n = 4, time = 10 ms

Find: Time when n = 16

Solution:

Step 1: T(n) = c × n²

Step 2: Find c:

10 = c × 4²

10 = c × 16

c = 10/16 = 5/8

Step 3: Calculate T(16):

T(16) = (5/8) × 16² = (5/8) × 256 = 160 ms

Answer: 160 milliseconds

Example 3: O(log₂ n) Algorithm

Given:

  • Algorithm runs in O(log₂ n) time
  • For n = 16, time = 28 ms

Find: Time when n = 64

Solution:

Step 1: T(n) = c × log₂(n)

Step 2: Find c:

28 = c × log₂(16)

28 = c × 4

c = 7

Step 3: Calculate T(64):

T(64) = 7 × log₂(64) = 7 × 6 = 42 ms

Answer: 42 milliseconds

Example 4: Finding Input Size

Given:

  • Algorithm runs in O(n²) time
  • For n = 10, time = 30 ms

Find: Input size n when time = 120 ms

Solution:

Step 1: T(n) = c × n²

Step 2: Find c:

30 = c × 10²

30 = c × 100

c = 3/10

Step 3: Solve for n when T(n) = 120:

120 = (3/10) × n²

n² = 120 × 10/3 = 400

n = 20

Answer: n = 20

Practice Examples

Practice 1: Simple Loop

int count = 0;
for (int i = 0; i < n; i++) {
    count++;
}

Polynomial: f(n) = n

Big O: O(n)

Practice 2: Nested Loop with Variable Inner

for (int i = 0; i < n; i++) {
    for (int j = i; j < n; j++) {
        sum += array[i][j];
    }
}

Analysis:

i=0: inner loop runs n times

i=1: inner loop runs n-1 times

i=2: inner loop runs n-2 times

Total: n + (n-1) + (n-2) + ... + 1 = n(n+1)/2

Polynomial: f(n) = n²/2 + n/2

Big O: O(n²)

Practice 3: Three Nested Loops

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        for (int k = 0; k < n; k++) {
            cube[i][j][k] = i + j + k;
        }
    }
}

Analysis: n × n × n

Polynomial: f(n) = n³

Big O: O(n³)

Practice 4: Loop with Division

int i = 1;
while (i < n) {
    System.out.println(i);
    i = i * 2;
}

Analysis:

i starts at 1, doubles each iteration: 1, 2, 4, 8, ..., 2^k

Loop ends when 2^k >= n, so k = log₂(n)

Polynomial: f(n) = log₂(n)

Big O: O(log n)

Practice 5: Mixed Loops

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        matrix[i][j] = 0;
    }
}

for (int k = 0; k < n; k++) {
    sum += array[k];
}

Analysis:

First nested loop: n²

Second loop: n

Total (add them): n² + n

Polynomial: f(n) = n² + n

Big O: O(n²)

Practice 6: Array Search

boolean found = false;
for (int i = 0; i < n && !found; i++) {
    if (array[i] == target) {
        found = true;
    }
}

Analysis:

Loop runs at most n times

Each iteration: 1 operation

Polynomial: f(n) = n

Big O: O(n)

Practice 7: Triangular Pattern

for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++) {
        System.out.print("*");
    }
    System.out.println();
}

Analysis:

i=1: inner runs 1 time

i=2: inner runs 2 times

i=n: inner runs n times

Total: 1 + 2 + 3 + ... + n = n(n+1)/2

Polynomial: f(n) = n²/2 + n/2

Big O: O(n²)

Practice 8: Constant Time Operations

int first = array[0];
int last = array[n-1];
int mid = array[n/2];
int sum = first + last + mid;

Analysis:

4 simple operations (no loops)

Operations don't depend on n

Polynomial: f(n) = 4

Big O: O(1)

Practice 9: Decreasing Loop

for (int i = n; i > 0; i--) {
    sum += array[i-1];
}

Analysis:

Loop runs from n down to 1: n iterations

Each iteration: 1 operation

Polynomial: f(n) = n

Big O: O(n)

Practice 10: Step by 2

for (int i = 0; i < n; i += 2) {
    process(array[i]);
}

Analysis:

Loop runs: 0, 2, 4, 6, ..., n-2

Total iterations: n/2

Polynomial: f(n) = n/2

Big O: O(n)

Note: Constants are dropped in Big O

Practice 11: Nested with Different Bounds

for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        matrix[i][j] = i + j;
    }
}

Analysis:

Outer loop: n times

Inner loop: m times

Total: n × m

Polynomial: f(n,m) = n × m

Big O: O(n × m)

Note: Two variables in complexity

Practice 12: Half Array Processing

for (int i = 0; i < n/2; i++) {
    for (int j = 0; j < n/2; j++) {
        sum += matrix[i][j];
    }
}

Analysis:

Outer loop: n/2 times

Inner loop: n/2 times

Total: (n/2) × (n/2) = n²/4

Polynomial: f(n) = n²/4

Big O: O(n²)

Practice 13: Reverse Triangular

for (int i = n; i > 0; i--) {
    for (int j = 0; j < i; j++) {
        System.out.print("*");
    }
}

Analysis:

i=n: inner runs n times

i=n-1: inner runs n-1 times

i=1: inner runs 1 time

Total: n + (n-1) + ... + 1 = n(n+1)/2

Polynomial: f(n) = n²/2 + n/2

Big O: O(n²)

Practice 14: Division by 3

int i = n;
while (i > 1) {
    System.out.println(i);
    i = i / 3;
}

Analysis:

i: n, n/3, n/9, n/27, ..., 1

After k iterations: n/3^k = 1

Solving: k = log₃(n)

Polynomial: f(n) = log₃(n)

Big O: O(log n)

Practice 15: Multiple Sequential Loops

for (int i = 0; i < n; i++) {
    array[i] = i;
}

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        matrix[i][j] = i * j;
    }
}

for (int i = 0; i < n; i++) {
    sum += array[i];
}

Analysis:

First loop: n

Second loop (nested): n²

Third loop: n

Total: n + n² + n = n² + 2n

Polynomial: f(n) = n² + 2n

Big O: O(n²)

Practice 16: Time Calculation Problem 1

Given:

  • Algorithm runs in O(n) time
  • For n = 50, time = 5 ms

Find: Time when n = 200

T(n) = c × n

5 = c × 50

c = 1/10

T(200) = (1/10) × 200 = 20 ms

Answer: 20 milliseconds

Practice 17: Time Calculation Problem 2

Given:

  • Algorithm runs in O(n²) time
  • For n = 5, time = 25 ms

Find: Time when n = 20

T(n) = c × n²

25 = c × 5²

25 = c × 25

c = 1

T(20) = 1 × 20² = 400 ms

Answer: 400 milliseconds

Practice 18: Time Calculation Problem 3

Given:

  • Algorithm runs in O(log₂ n) time
  • For n = 32, time = 10 ms

Find: Time when n = 256

T(n) = c × log₂(n)

10 = c × log₂(32)

10 = c × 5

c = 2

T(256) = 2 × log₂(256) = 2 × 8 = 16 ms

Answer: 16 milliseconds

Practice 19: Finding Input Size

Given:

  • Algorithm runs in O(n²) time
  • For n = 20, time = 80 ms

Find: What is n when time = 320 ms?

T(n) = c × n²

80 = c × 20²

80 = c × 400

c = 1/5

320 = (1/5) × n²

n² = 320 × 5 = 1600

n = 40

Answer: n = 40

Practice 20: Polynomial to Big O

Convert the following polynomials to Big O notation:

  1. f(n) = 5n³ + 2n² + 100
  2. f(n) = 1000n + 500
  3. f(n) = n² log(n) + 3n²
  4. f(n) = 2^n + n³
  5. f(n) = 15

Solutions:

  1. f(n) = 5n³ + 2n² + 100 → O(n³)
  2. f(n) = 1000n + 500 → O(n)
  3. f(n) = n² log(n) + 3n² → O(n² log n)
  4. f(n) = 2^n + n³ → O(2^n) (exponential dominates)
  5. f(n) = 15 → O(1)

Challenging Problems

Challenge 1: Logarithmic with Nested Loop

for (int i = 1; i < n; i = i * 2) {
    for (int j = 0; j < n; j++) {
        process(i, j);
    }
}

Analysis:

Outer loop: runs log₂(n) times (i doubles)

Inner loop: runs n times for each outer iteration

Polynomial: f(n) = n × log₂(n)

Big O: O(n log n)

Challenge 2: Complex Condition

int i = 0, j = 0;
while (i < n) {
    while (j < n && array[i][j] == 1) {
        j++;
    }
    i++;
}

Analysis:

Important: j is NOT reset in outer loop!

i increments n times total

j increments at most n times total (never reset)

Maximum total increments: 2n

Polynomial: f(n) = 2n

Big O: O(n)

Challenge 3: Recursive Doubling

void mystery(int n) {
    if (n <= 1) return;

    for (int i = 0; i < n; i++) {
        System.out.println(i);
    }

    mystery(n/2);
    mystery(n/2);
}

Analysis:

Recurrence: T(n) = 2T(n/2) + n

Each level does n work total

Number of levels: log₂(n)

Total: n × log₂(n)

Big O: O(n log n)

Challenge 4: Square Root Loop

for (int i = 1; i * i <= n; i++) {
    for (int j = 0; j < n; j++) {
        process(i, j);
    }
}

Analysis:

Outer loop: runs while i² ≤ n, so i ≤ √n

Outer loop runs √n times

Inner loop runs n times

Polynomial: f(n) = √n × n = n^(3/2)

Big O: O(n√n) or O(n^1.5)

Challenge 5: Factorial-like Pattern

for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++) {
        for (int k = 1; k <= j; k++) {
            sum++;
        }
    }
}

Analysis:

For i=1: 1 operation

For i=2: 1+2 = 3 operations

For i=3: 1+2+3 = 6 operations

For i=n: 1+2+3+...+n = n(n+1)/2 operations

Total: sum of all triangular numbers = n(n+1)(n+2)/6

Polynomial: f(n) ≈ n³/6

Big O: O(n³)

Challenge 6: Time Calculation Problem

Given:

  • Algorithm A runs in O(n log n) time
  • For n = 8, time = 24 ms
  • Algorithm B runs in O(n²) time
  • For n = 10, time = 50 ms

Question: Which algorithm is faster for n = 100?

Algorithm A:

24 = c × 8 × log₂(8) = c × 8 × 3

c = 24/24 = 1

T(100) = 1 × 100 × log₂(100) ≈ 100 × 6.64 = 664 ms

Algorithm B:

50 = c × 10² = c × 100

c = 0.5

T(100) = 0.5 × 100² = 0.5 × 10,000 = 5,000 ms

Algorithm A is faster! (664 ms vs 5,000 ms)
