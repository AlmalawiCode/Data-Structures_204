üìñ Introduction

Welcome to your first cybersecurity lab! In this lab, you will learn how to set up a professional penetration testing environment by installing Kali Linux as a Virtual Machine on Oracle VirtualBox. This is a fundamental skill that every cybersecurity professional must master.

üí°

Why Virtual Machines?

Virtual machines allow you to run multiple operating systems on a single computer safely. This is essential for:

  • Testing security tools without affecting your main system
  • Creating isolated lab environments
  • Learning different operating systems
  • Taking snapshots to restore previous states

  • Successfully boot and log into your Kali Linux virtual machine
  • Navigate the basic Kali Linux interface

Important: Enable Virtualization

Virtualization must be enabled in your computer's BIOS/UEFI settings. To check on Windows:

  1. Open Task Manager (Ctrl + Shift + Esc)
  2. Go to the Performance tab
  3. Click on CPU
  4. Look for "Virtualization: Enabled"

üì• Part 2: Downloading Kali Linux

Now that VirtualBox is installed, let's download the Kali Linux virtual machine image.

Step 1: Go to Kali Download Page

  1. Open your web browser
  2. Navigate to the official Kali Linux download page:

    üêâ Kali Linux Downloads - Virtual Machines

Step 2: Select VirtualBox Image

  1. On the download page, look for the "Virtual Machines" section
  2. Find the VirtualBox option
  3. Click the Download button for the 64-bit version
File Information Details
Format .7z (7-Zip compressed archive)
Contains A .vbox file + .vmdk virtual disk
Size ~3-4 GB (compressed)
Architecture 64-bit (amd64)
‚è≥

Download Time

The download may take 10-30 minutes depending on your internet speed. Please be patient!

Step 3: Verify Download (Recommended)

üîí

Security Best Practice

Always verify downloaded files to ensure they haven't been tampered with. Compare the SHA256 checksum shown on the download page with your file:

Windows (PowerShell):

Get-FileHash .\kali-linux-*-virtualbox-amd64.7z -Algorithm SHA256

macOS (Terminal):

shasum -a 256 ~/Downloads/kali-linux-*-virtualbox-amd64.7z

Step 4: Extract the Archive

The downloaded file is a .7z compressed archive. You need to extract it before importing into VirtualBox.

  1. If you don't have 7-Zip installed:
    • Windows: Download from 7-zip.org and install it
    • macOS: Install Keka or use: brew install p7zip
  2. Right-click the downloaded .7z file
  3. Select 7-Zip → Extract Here (or Extract to "kali-linux-...")
  4. Wait for extraction to complete — this will create a folder containing the VM files
üìÇ

After Extraction

You will find a folder containing a .vbox file and a .vmdk file. The .vbox file is the VirtualBox configuration, and the .vmdk is the virtual hard disk.

üì¶ Part 3: Adding Kali Linux to VirtualBox

Now we will add the extracted Kali Linux image as a virtual machine in VirtualBox.

Step 1: Open VirtualBox

  1. Launch Oracle VM VirtualBox from:
    • Windows: Start Menu ‚Üí VirtualBox
    • macOS: Applications ‚Üí VirtualBox

Step 2: Add the Kali Linux VM

  1. In VirtualBox, go to Machine → Add
    Or use keyboard shortcut: Ctrl + A (Windows) / Cmd + A (macOS)
  2. Navigate to the extracted folder from the previous step
  3. Select the .vbox file (e.g., kali-linux-*-virtualbox-amd64.vbox)
  4. Click Open

Step 3: Review Virtual Machine Settings

You will see the VM settings. Review and optionally modify them:

Setting Default Recommendation
Name Kali Linux Keep or customize
CPU 2 cores Increase to 4 if possible
RAM 2048 MB Increase to 4096 MB if possible
Network NAT Keep as NAT

Performance Tip

If your computer has 16 GB or more RAM, allocate 4 GB (4096 MB) to Kali for better performance.

Step 4: Verify the VM is Added

  1. "Kali Linux" should now appear in your VirtualBox Manager
  2. The VM is ready — no import wait time needed since the files are already extracted

üîß Troubleshooting

If you encounter issues, check these common problems and solutions:

‚ùå "VT-x is not available" Error

Problem: Virtualization is disabled in BIOS/UEFI.

Solution:

  1. Restart your computer
  2. Enter BIOS/UEFI setup (press F2, F10, F12, or Del during boot)
  3. Find virtualization settings (VT-x, AMD-V, SVM, or Virtualization Technology)
  4. Enable virtualization
  5. Save and exit BIOS
‚ùå Black Screen After Boot

Problem: Display settings issue.

Solution:

  1. Power off the VM
  2. Go to Settings ‚Üí Display
  3. Increase Video Memory to 128 MB
  4. Disable 3D Acceleration
  5. Try starting again
‚ùå Slow Performance

Problem: Insufficient resources allocated to VM.

Solution:

  1. Power off the VM
  2. Go to Settings ‚Üí System
  3. Increase RAM to 4096 MB
  4. Go to Settings ‚Üí Processor
  5. Increase CPU cores to 2-4
‚ùå Network Not Working

Problem: Network adapter configuration issue.

Solution:

  1. Power off the VM
  2. Go to Settings ‚Üí Network
  3. Ensure Adapter 1 is enabled
  4. Set "Attached to" as NAT
  5. Click OK and start the VM
‚ùå Add/Extract Failed

Problem: Corrupted download, failed extraction, or insufficient disk space.

Solution:

  1. Delete the downloaded .7z file and extracted folder
  2. Re-download from the official Kali website
  3. Verify the SHA256 checksum
  4. Ensure you have at least 50 GB free disk space
  5. Make sure 7-Zip is installed correctly for extraction
