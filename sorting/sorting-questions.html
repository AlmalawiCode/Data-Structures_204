Question: Given the initial array, apply Selection Sort to sort in ascending order (smallest → largest). Scan from left to right starting at index 0 each loop, find the minimum, and swap it into position. Drag the tiles to fill in the full array state after each loop.

Selection Sort — Algorithm Contract

Idea: Repeatedly find the minimum element from the unsorted portion and swap it into its correct position.

  • Pass 1: Scan all n elements → find min → swap to index 0
  • Pass 2: Scan indices 1..n−1 → find min → swap to index 1
  • After n−1 passes the array is fully sorted
Best O(n²)
Average O(n²)
Worst O(n²)
Space O(1)
Stable? No
Question: Given the initial array, apply Bubble Sort to sort in ascending order (smallest → largest). Each loop compares adjacent pairs starting from index 0, swapping out-of-order elements. Drag the tiles to fill in the full array state after each loop.

Bubble Sort — Algorithm Contract

Idea: Repeatedly compare adjacent pairs and swap if out of order. The largest unsorted element "bubbles up" each pass.

  • Pass 1: Compare a[0]&a[1], a[1]&a[2], … → largest at end
  • Pass 2: Repeat for 0..n−2 → second-largest settles
  • Early exit: stop if no swaps occur in a pass
Best O(n)
Average O(n²)
Worst O(n²)
Space O(1)
Stable? Yes
Question: Given the initial array, apply Insertion Sort to sort in ascending order (smallest → largest). Start with the element at index 1 as the first key; shift larger elements rightward to insert each key in its correct position. Drag the tiles to fill in the full array state after each step.

Insertion Sort — Algorithm Contract

Idea: Build a sorted sub-array one element at a time by inserting each new element into its correct position.

  • Step 1: Take a[1] as key → shift larger elements right → insert
  • Step 2: Take a[2] → shift right as needed → insert
  • After n−1 steps the full array is sorted
Best O(n)
Average O(n²)
Worst O(n²)
Space O(1)
Stable? Yes
Question: Given the initial array, apply Merge Sort to sort in ascending order (smallest → largest). The array is recursively split in half, then sorted halves are merged by always picking the smaller front element. Each row shows the full array state after one merge operation completes. Merge Sort — Algorithm Contract

Idea: Divide-and-conquer. Recursively split until 1-element sub-arrays, then merge sorted halves by comparing front elements.

  • Divide: split [L..R] into [L..mid] and [mid+1..R] recursively
  • Merge: always pick the smaller front element from each half
  • Each row shows the full array state after one merge operation
Best O(n log n)
Average O(n log n)
Worst O(n log n)
Space O(n)
Stable? Yes
Question: Given the initial array, apply Quick Sort to sort in ascending order (smallest → largest), based on the Hoare Partition Scheme. Assume the first element of each sub-array is the pivot. Each row shows the full array state after one partition completes. Drag the tiles to fill in the array state after each partition.

Quick Sort — Algorithm Contract

Idea: Divide-and-conquer. Pick the first element as pivot (Hoare Partition Scheme), partition using two pointers that scan inward and swap, then recurse on each side.

  • Pivot = first element A[l] of the current sub-array range (pivot selection is flexible — random or other strategies are valid too)
  • Partition: pointer i scans right (past elements ≤ pivot), pointer j scans left (past elements > pivot); swap when both stop; place pivot at j when i ≥ j
  • Each row shows the full array after one partition completes
Best O(n log n)
Average O(n log n)
Worst O(n²)
Space O(log n)
Stable? No
