1 · Selection Sort

Core Idea: Divide the array into a sorted part (on the left) and an unsorted part (on the right). In each pass, find the minimum (or maximum) element in the unsorted part and move it to the boundary.

How It Works (Ascending)

  1. Assume the first element of the unsorted part is the minimum.
  2. Scan the rest of the unsorted part, updating the minimum if a smaller element is found.
  3. After the scan, swap the found minimum with the element at the boundary.
  4. Move the boundary one step to the right. Repeat until the entire array is sorted.

Algorithm Analysis — Why O(n²)?

Selection sort uses two nested loops. Let us count exactly how many comparisons it makes:

Step 1 — Count comparisons per pass:

T(n) = n + (n−1) + (n−2) + (n−3) + … + 3 + 2 + 1

Pass 1 scans n elements  |  Pass 2 scans n−1  |  …  |  Pass n scans 1

Write the sum forward and backward, then add column by column — each pair sums to (n+1), and there are n pairs:

T(n) =   1   +   2   +   3   + … + (n−1) +   n  
T(n) =   n   + (n−1) + (n−2) + … +   2   +   1  
2·T(n) = (n+1) + (n+1) + (n+1) + … + (n+1) + (n+1)

2 · T(n) = n × (n+1)    ←   n pairs, each equal to (n+1)

Step 2 — Divide both sides by 2:

T(n)  =  n  (n+1)2

Step 3 — Expand the numerator (two terms):

T(n)  =  n² + n2  =  12n²  +  12n

The dominant term is ½n²; the lower-order term ½n is negligible for large n.

∴   T(n) ∈ O(n²)

Drop the constant ½ and the lower-order term ½n. Only n² matters.

Concrete example — n = 6:

T(6) = 1 + 2 + 3 + 4 + 5 + 6
T(6) = 6 + 5 + 4 + 3 + 2 + 1
2·T(6) = 7 + 7 + 7 + 7 + 7 + 7

2 · T(6) = 6 × 7 = 42

T(6)  =  422  =  21 comparisons  ✓

Verify:   n(n+1)2  =  6 × 72  =  422  =  21  ✓

Important: Selection sort always makes exactly n(n+1)/2 comparisons, regardless of whether the array is already sorted or completely reversed. The number of swaps may vary (0 to n−1), but comparisons never do.
Selection Sort — Characteristics:
  • Always performs the same number of comparisons regardless of input order
  • Not Adaptive — does not benefit from partially sorted data
  • Time Complexity: O(n²) in all cases (best, average, and worst)
  • The number of swaps varies (0 to n−1), but comparisons are always exactly n(n−1)/2

Java Implementation

Selection Sort — Java▼ 
public static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {        // boundary position
        int minIdx = i;                            // assume min is at boundary
        for (int j = i + 1; j < n; j++) {        // scan unsorted part
            if (arr[j] < arr[minIdx]) {
                minIdx = j;                        // update min index
            }
        }
        if (minIdx != i) {                         // swap if needed
            int temp = arr[i];
            arr[i]      = arr[minIdx];
            arr[minIdx] = temp;
        }
    }
}

2 · Insertion Sort

Core Idea: Like sorting a hand of playing cards. Start with one card (trivially sorted). Pick the next card and insert it into the correct position among the already-sorted cards. Repeat.

How It Works (Ascending)

  1. The first element alone forms a sorted sub-array of size 1 — trivially sorted!
  2. Take the next element (the key) from the unsorted part.
  3. Compare the key with elements in the sorted part, scanning right to left.
  4. Shift each element that is greater than the key one position to the right.
  5. Drop the key into the gap created. The sorted sub-array grows by one.
  6. Repeat from step 2 until all elements have been inserted.

Algorithm Analysis

Best Case (already sorted): O(n)

Each key is already in place → inner loop runs 0 times → only n−1 comparisons total

Worst Case (reverse sorted): O(n²)

T(n) = 1 + 2 + 3 + … + (n−1) = n(n−1)/2 — same as selection sort

Average Case: O(n²)

Key insight: Insertion sort can stop early (break out of the inner loop) as soon as it finds the correct position. This makes it much faster than Selection Sort when the data is nearly sorted!
Insertion Sort — Characteristics:
  • Naturally Adaptive — no extra optimization flag needed; the inner loop stops as soon as the position is found
  • Best Case: O(n) — already sorted; each element needs only 1 comparison
  • Worst Case: O(n²) — reverse sorted data
  • Highly efficient for small arrays or nearly sorted data; often outperforms both Selection Sort and Bubble Sort in practice

Java Implementation

Insertion Sort — Java▼ 
public static void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];         // element to be inserted
        int j   = i - 1;
        // shift elements greater than key to the right
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];   // shift right
            j--;
        }
        arr[j + 1] = key;          // insert key at correct position
    }
}

📌 Important Observation for Students

Try pressing Next when the key is 82 (the last element in our array). Notice that only one comparison is made, and the line immediately moves right — no swap at all.

Why? Because everything to the left of the divider line is guaranteed to be sorted. The last element in the sorted portion is the largest among them. So if the key is already larger than that element, it must be larger than all sorted elements — no further comparisons are needed.

Data Condition Comparisons per key Total comparisons
Fully sorted — every key is already in place 1 each n − 1 → O(n)
Nearly sorted — most keys need few steps 1 – 3 each Close to O(n)
Reverse sorted — every key walks to position 0 i each n(n−1)/2 → O(n²)

Conclusion: Selection Sort always makes exactly n(n−1)/2 comparisons no matter what the data looks like. Insertion Sort adapts — on nearly-sorted data it can be dramatically faster. This is its greatest strength.

3 · Bubble Sort

Core Idea: Compare adjacent pairs and swap them if they are in the wrong order. After each pass, the largest unsorted element has "bubbled up" to its correct position at the end.

Part A — Basic Bubble Sort

In its basic form, Bubble Sort runs all n−1 passes regardless of input order:

  1. Start at the beginning of the array.
  2. Compare each adjacent pair (arr[j], arr[j+1]). If arr[j] > arr[j+1], swap them.
  3. After one full pass, the largest element has "bubbled" to the last position — correctly placed.
  4. Repeat for all n−1 passes — always runs every pass, even if the array is already sorted.
Basic Bubble Sort — Characteristics:
  • Time Complexity: O(n²) in all cases — always runs all n−1 passes
  • Not Adaptive — cannot detect if the array is already sorted
  • Identical to Selection Sort in performance: both always make exactly n(n−1)/2 comparisons

Part B — Optimized Bubble Sort (Early Exit)

A single boolean flag (swapped) transforms Bubble Sort into an adaptive algorithm — this is considered a "smart optimization":

  1. At the start of each pass, set swapped = false.
  2. If any swap occurs during the pass, set swapped = true.
  3. After the pass: if swapped == false, no elements were out of order → the array is already sorted → stop immediately.
Optimized Bubble Sort — Characteristics:
  • Adaptive: YES — the "no swaps" check detects sorted data and exits early
  • Best Case: O(n) — already sorted data triggers early exit after just one pass
  • Worst Case: O(n²) — reverse sorted data (same as basic version)
Why is it called "Bubble Sort"?
Watch the simulation: in each pass, the largest remaining element "bubbles up" to the right end, like a bubble rising through water — it keeps swapping with its right neighbour until it reaches the surface (its correct position).

Algorithm Analysis

Worst Case: O(n²)

Reverse-sorted input — every pair needs swapping every pass

T(n) = (n−1) + (n−2) + … + 1 = n(n−1)/2

Best Case (with early-exit optimisation): O(n)

Already sorted — first pass finds no swaps → stops immediately after n−1 comparisons

Optimized Bubble Sort: There is a well-known variant that adds a swapped flag — if a full pass completes with no swaps, the array is already sorted and the algorithm terminates immediately, giving O(n) in the best case. If you would like to explore this further, search "optimized bubble sort" online.

Java Implementation

Bubble Sort — Java (with early-exit optimisation)▼ 
public static void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int pass = 0; pass < n - 1; pass++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - pass; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp  = arr[j];
                arr[j]     = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break;  // already sorted — early exit
    }
}

4 · Insertion Sort vs Selection Sort — Does Data Matter?

A student asks: "Professor, if both Selection Sort and Insertion Sort are O(n²), which one is actually faster?"

Professor replies: "Excellent question! They have the same worst-case complexity, but in practice Insertion Sort can be much faster when the data is nearly sorted. Let me show you why."

The Key Difference

Selection Sort

Always scans the entire remaining unsorted part to find the minimum.
Number of comparisons = n(n−1)/2, regardless of the input.

Insertion Sort

Can stop early once the correct insertion position is found.
For nearly-sorted data, almost every key stops after just 1–2 comparisons!
