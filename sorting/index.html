1 · What is Sorting?

Sorting is one of the most fundamental operations in computing. Every time you see a list ordered — from the cheapest flight to the most expensive, from newest email to oldest, from "A" to "Z" — sorting is at work behind the scenes.

It is so common that computer scientists have spent decades inventing and refining algorithms to do it as fast as possible. In this module you will learn how those algorithms work and why some are faster than others.

3 · Two Sorting Families

Sorting algorithms are a hot research topic. How fast can we sort? This question has driven decades of innovation. In this module we study two families:

O(n²)
Quadratic Sorting

Selection · Insertion · Bubble
Simple to understand; slow on large data

O(n log n)
Logarithmic Sorting

Quick Sort · Merge Sort
More complex; efficient on large data

Why does complexity matter?

When we say an algorithm is O(n²), its number of operations grows as the square of the input size. Double the data → four times the work. For small arrays this is fine; for millions of records it becomes a bottleneck. The graph below shows how dramatically the curves diverge as n grows.

Example (n = 1000 elements):
  • O(n) → ~1,000 operations
  • O(n log n) → ~10,000 operations
  • O(n²) → ~1,000,000 operations — 1000× more than O(n log n)!
The takeaway: For everyday use with small datasets both families work fine. But when the data grows large — millions of contacts, billions of web pages — only O(n log n) algorithms are practical.
