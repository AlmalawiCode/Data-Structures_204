1. ما هو المكدس؟

التعريف: المكدس (Stack) هو هيكل بيانات يخزن المعلومات في شكل مجموعة خطية حيث تُضاف العناصر وتُزال من طرف واحد فقط يُسمى القمة (top). ويتبع مبدأ LIFO (الأخير دخولاً، الأول خروجاً).

أمثلة من الحياة اليومية

🍽️ كومة الأطباق

تُضاف الأطباق من الأعلى وتُزال من الأعلى. آخر طبق يُوضع هو أول طبق يُزال.

📚 كومة الكتب

تُكدَّس الكتب عمودياً. تأخذ الكتاب العلوي أولاً.

↩️ سجل المتصفح

يأخذك زر الرجوع إلى آخر صفحة زرتها (LIFO).

↶ عملية التراجع (Undo)

آخر إجراء تم تنفيذه هو أول إجراء يُتراجع عنه.

الخصائص الرئيسية للمكدس

  • هيكل بيانات خطي: العناصر مرتبة بصورة متتالية
  • متجانس: جميع العناصر من نوع البيانات نفسه
  • وصول من طرف واحد: يمكن الوصول إلى العناصر من القمة فقط
  • سياسة LIFO: الأخير دخولاً، الأول خروجاً - العنصر المضاف مؤخراً يُزال أولاً

2. المكدس كنوع مجرد للبيانات (ADT)

لماذا يُعدّ المكدس نوعاً مجرداً للبيانات؟

يُعدّ المكدس نوعاً مجرداً للبيانات (ADT) لأنه يُعرّف سلوك هيكل البيانات من خلال عملياته، دون تحديد كيفية تطبيق هذه العمليات داخلياً.

البنية المنطقية مقابل البنية المادية

هيكل البيانات المنطقي

المكدس هو هيكل بيانات منطقي - يصف كيفية تنظيم البيانات والوصول إليها (سلوك LIFO)، لكن لا يحدد كيفية تخزينها في الذاكرة.

التطبيق المادي

لاستخدام المكدس فعلياً، يجب بناؤه فوق هيكل بيانات مادي:

  • التطبيق باستخدام المصفوفة - يستخدم ذاكرة متجاورة
  • التطبيق باستخدام القائمة المرتبطة - يستخدم تخصيصاً ديناميكياً للذاكرة

العمليات المجردة

تبقى عمليات المكدس ثابتة بغض النظر عن التطبيق الأساسي:

العملية الوصف التطبيق بالمصفوفة التطبيق بالقائمة المرتبطة
push() إضافة عنصر إلى القمة arr[++top] = element إضافة عقدة في البداية
pop() إزالة عنصر القمة return arr[top--] إزالة عقدة من البداية
top()/peek() عرض عنصر القمة return arr[top] return head.data
isEmpty() التحقق من الفراغ return top == -1 return head == null
isFull() التحقق من الامتلاء return top == size-1 لا ينطبق (ديناميكي)
نقطة أساسية: يمكنك تغيير التطبيق المادي (مصفوفة ↔ قائمة مرتبطة)، لكن العمليات المجردة وسلوكها يبقيان ثابتَين. هذا هو جوهر التجريد!

3. عمليات المكدس

3.1 push(element) - الإضافة

الغرض: إضافة عنصر إلى قمة المكدس

الخطوات:

  1. التحقق من امتلاء المكدس (لتطبيق المصفوفة)
  2. زيادة مؤشر القمة
  3. إدراج العنصر في موضع القمة

التعقيد الزمني: O(1)

3.2 pop() - الإزالة

الغرض: إزالة وإرجاع عنصر القمة من المكدس

الخطوات:

  1. التحقق من فراغ المكدس
  2. استرداد العنصر من موضع القمة
  3. تقليل مؤشر القمة
  4. إرجاع العنصر المسترد

التعقيد الزمني: O(1)

3.3 top() / peek() - المعاينة

الغرض: إرجاع عنصر القمة دون إزالته

الخطوات:

  1. التحقق من فراغ المكدس
  2. إرجاع العنصر في موضع القمة

التعقيد الزمني: O(1)

3.4 isEmpty() - فارغة؟

الغرض: التحقق مما إذا كان المكدس فارغاً

يُرجع: true إذا لم يكن في المكدس عناصر، false في غير ذلك

التعقيد الزمني: O(1)

3.5 isFull() - ممتلئة؟

الغرض: التحقق مما إذا كان المكدس ممتلئاً (أساساً لتطبيق المصفوفة)

يُرجع: true إذا لم يستطع المكدس قبول مزيد من العناصر، false في غير ذلك

ملاحظة: لا ينطبق على تطبيق القائمة المرتبطة (حجم ديناميكي)

التعقيد الزمني: O(1)

5. تطبيق المكدس بلغة Java

5.1 التطبيق باستخدام المصفوفة (حجم ثابت)

باستخدام مصفوفة ذات حجم ثابت مع سعة قصوى تُحدَّد عند التهيئة. بعد امتلاء المصفوفة، لا يمكن إضافة مزيد من العناصر إلا بعد حذف عناصر.

مهم: للمصفوفات في Java حجم ثابت. لا يمكن زيادة حجم المصفوفة بعد إنشائها. يجب تحديد الحجم عند التهيئة ويبقى ثابتاً طوال عمر المصفوفة.
📄 StackUsingArray.java (Fixed Size) ▼
public class StackUsingArray<T> { private T[] stack; private int top; private int maxSize; // Constructor - Array size MUST be specified and is FIXED @SuppressWarnings("unchecked") public StackUsingArray(int size) { maxSize = size; stack = (T[]) new Object[maxSize]; // Fixed size array top = -1; // Stack is initially empty } // Push operation - O(1) public boolean push(T element) { if (isFull()) { System.out.println("Stack Overflow! Cannot push " + element); return false; } stack[++top] = element; // Increment top, then add element return true; } // Pop operation - O(1) public T pop() { if (isEmpty()) { System.out.println("Stack Underflow! Stack is empty"); return null; } T element = stack[top]; stack[top--] = null; // Remove reference, then decrement top return element; } // Peek/Top operation - O(1) public T top() { if (isEmpty()) { System.out.println("Stack is empty"); return null; } return stack[top]; } // Check if stack is empty - O(1) public boolean isEmpty() { return top == -1; } // Check if stack is full - O(1) public boolean isFull() { return top == maxSize - 1; } // Get current size - O(1) public int size() { return top + 1; } // Display stack contents public void display() { if (isEmpty()) { System.out.println("Stack is empty"); return; } System.out.println("Stack (top to bottom): "); for (int i = top; i >= 0; i--) { System.out.println(" " + stack[i]); } } } // Example Usage public class Main { public static void main(String[] args) { StackUsingArray<Integer> stack = new StackUsingArray<>(5); // Fixed size of 5 stack.push(10); stack.push(20); stack.push(30); stack.display(); System.out.println("Top element: " + stack.top()); System.out.println("Popped: " + stack.pop()); stack.display(); // Try to exceed capacity stack.push(40); stack.push(50); stack.push(60); // This will cause Stack Overflow! } }

5.2 التطبيق باستخدام القائمة المرتبطة

استخدام القائمة المرتبطة يوفر حجماً ديناميكياً حقيقياً دون قيود على السعة.

📄 StackUsingLinkedList.java ▼
// Node class for Linked List class Node<T> { T data; Node<T> next; public Node(T data) { this.data = data; this.next = null; } } public class StackUsingLinkedList<T> { private Node<T> top; private int size; // Constructor public StackUsingLinkedList() { top = null; size = 0; } // Push operation - O(1) public void push(T element) { Node<T> newNode = new Node<>(element); newNode.next = top; top = newNode; size++; } // Pop operation - O(1) public T pop() { if (isEmpty()) { System.out.println("Stack Underflow! Stack is empty"); return null; } T data = top.data; top = top.next; size--; return data; } // Peek/Top operation - O(1) public T top() { if (isEmpty()) { System.out.println("Stack is empty"); return null; } return top.data; } // Check if stack is empty - O(1) public boolean isEmpty() { return top == null; } // Note: isFull() is not needed for linked list implementation // as it grows dynamically // Get current size - O(1) public int size() { return size; } // Display stack contents public void display() { if (isEmpty()) { System.out.println("Stack is empty"); return; } System.out.println("Stack (top to bottom): "); Node<T> current = top; while (current != null) { System.out.println(" " + current.data); current = current.next; } } } // Example Usage public class Main { public static void main(String[] args) { StackUsingLinkedList<String> stack = new StackUsingLinkedList<>(); stack.push("Java"); stack.push("Python"); stack.push("C++"); stack.display(); System.out.println("Top element: " + stack.top()); System.out.println("Popped: " + stack.pop()); stack.display(); } }

مقارنة: تطبيق المصفوفة مقابل القائمة المرتبطة

الجانب يعتمد على المصفوفة يعتمد على القائمة المرتبطة
الذاكرة حجم ثابت، ذاكرة متجاورة حجم ديناميكي، غير متجاور
حد الحجم له سعة قصوى لا حد ثابت (حتى ينفد الذاكرة)
عبء الذاكرة أقل (عناصر فقط) أعلى (عناصر + مؤشرات)
أداء التخزين المؤقت أفضل (متجاور) أسوأ (متفرق)
عملية isFull() ضرورية وذات معنى لا تنطبق

6. تطبيقات المكدس

مهم: المكدسات هياكل بيانات أساسية ذات تطبيقات لا تُعدّ في علوم الحاسب. التطبيقات شبه لا محدودة وتعتمد على مجال المشكلة. سنركز على أحد أهم التطبيقات: تقييم التعابير الحسابية.

تطبيقات المكدس الشائعة

  • تقييم التعابير وتحويلها (الترميز الداخلي Infix، اللاحق Postfix، السابق Prefix)
  • إدارة استدعاءات الدوال (مكدس الاستدعاء Call Stack في البرمجة)
  • عمليات التراجع/الإعادة (Undo/Redo) (محررات النصوص، برامج الرسومات)
  • سجل المتصفح (أزرار الرجوع والتقدم)
  • تحليل الصياغة (Syntax Parsing) (التحقق من الأقواس المتوازنة)
  • خوارزميات التراجع (Backtracking) (حل المتاهات، ألعاب الشطرنج)
  • إدارة الذاكرة (تخصيص ذاكرة المكدس)
  • اجتياز الشجرة والرسم البياني (DFS - البحث العميق أولاً)

تطبيق محوري: تقييم التعابير الحسابية

المشكلة

كيف يحل الحاسب التعابير الرياضية مثل 3 + 5 * 2 - 8 / 4؟

نحن كبشر نفهم أولوية العمليات ويمكننا تقييم التعابير المكتوبة بـالترميز الداخلي (Infix) (العمليات بين المعاملات). لكن الحاسب يعالج التعليمات بشكل تسلسلي من اليسار إلى اليمين، وهو يحتاج إلى طريقة منهجية لتقييم التعابير مع احترام أولوية العمليات.

6.1 أنواع التعابير الحسابية

الترميز الصيغة مثال
الترميز الداخلي (Infix) معامل عملية معامل A + B
الترميز اللاحق (Postfix) معامل معامل عملية A B +
الترميز السابق (Prefix) عملية معامل معامل + A B

مقارنة بالأمثلة

Infix (للإنسان) Postfix (للحاسب)
3 + 5 3 5 +
(3 + 5) * 2 3 5 + 2 *
3 + 5 * 2 3 5 2 * +
((3 + 5) * 2 - 8) / 4 3 5 + 2 * 8 - 4 /

6.2 لماذا الترميز اللاحق (Postfix)؟

الترميز اللاحق (المعروف أيضاً بالترميز البولندي العكسي - RPN) يُفضّله الحاسب لأنه:

  • لا حاجة إلى أقواس
  • لا حاجة إلى تذكر أولوية العمليات
  • يمكن تقييمه بمسح واحد من اليسار إلى اليمين
  • سهل التطبيق باستخدام المكدس
  • أكثر كفاءة للحسابات

6.3 أولوية العمليات والترابط

قبل تحويل التعابير، نحتاج إلى فهم أولوية العمليات:

العملية الأولوية الترابط
^ 3 (الأعلى) من اليمين إلى اليسار
*, /, % 2 من اليسار إلى اليمين
+, - 1 (الأدنى) من اليسار إلى اليمين
( ) تتجاوز الأولوية -

6.4 خوارزمية: تحويل Infix إلى Postfix

📋 الخطوات:

  1. أنشئ مكدساً فارغاً للعمليات
  2. أنشئ سلسلة نصية فارغة للمخرجات
  3. امسح التعبير الداخلي من اليسار إلى اليمين:
    • إذا كان معاملاً (A-Z, 0-9): أضفه مباشرة إلى المخرجات
    • إذا كان '(': ادفعه إلى المكدس
    • إذا كان ')': أخرج العمليات من المكدس إلى المخرجات حتى يُعثر على '(' (تجاهل القوسين)
    • إذا كان عملية:
      • بينما المكدس غير فارغ وقمة المكدس ليست '(' وقمة المكدس لها أولوية أعلى أو مساوية للعملية الممسوحة:
        • أخرج قمة المكدس إلى المخرجات
        • أعد التحقق من الشرط (كرر)
      • ادفع العملية الحالية إلى المكدس
  4. أخرج العمليات المتبقية من المكدس إلى المخرجات

6.5 خوارزمية: تقييم تعبير Postfix

📋 الخطوات:

  1. أنشئ مكدساً فارغاً للمعاملات
  2. امسح تعبير Postfix من اليسار إلى اليمين:
    • إذا كان معاملاً: ادفعه إلى المكدس
    • إذا كان عملية:
      • أخرج واحفظ في var1 (كان في القمة)
      • أخرج واحفظ في var2 (كان أسفل var1)
      • طبّق العملية كـ: result = var2 operator var1
      • مثال: لـ "5 3 -"
        المكدس يحتوي [5, 3] حيث 3 في القمة
        أخرج var1 = 3 (القمة)
        أخرج var2 = 5 (كان أسفلها)
        احسب: 5 - 3 = 2
      • ادفع النتيجة إلى المكدس
  3. القيمة النهائية في المكدس هي النتيجة

مثال يدوي: التحويل والتقييم

مثال: تحويل وتقييم 3 + 5 * 2

الخطوة 1: تحويل Infix إلى Postfix

الرمز الإجراء المكدس المخرجات
3 معامل → مخرجات [ ] 3
+ عملية → مكدس [ + ] 3
5 معامل → مخرجات [ + ] 3 5
* أولوية أعلى → مكدس [ +, * ] 3 5
2 معامل → مخرجات [ +, * ] 3 5 2
النهاية أخرج جميع العمليات [ ] 3 5 2 * +

نتيجة Postfix: 3 5 2 * +

الخطوة 2: تقييم تعبير Postfix

الرمز الإجراء المكدس
3 ادفع معاملاً [ 3 ]
5 ادفع معاملاً [ 3, 5 ]
2 ادفع معاملاً [ 3, 5, 2 ]
* أخرج 2 و 5، احسب 5*2=10، ادفع [ 3, 10 ]
+ أخرج 10 و 3، احسب 3+10=13، ادفع [ 13 ]

النتيجة النهائية: 13

9. الملخص

النقاط الأساسية

  • المكدس (Stack) هو هيكل بيانات LIFO (الأخير دخولاً، الأول خروجاً)
  • هو نوع مجرد للبيانات (ADT) - يُعرّف السلوك لا التطبيق
  • المكدس بنية منطقية يجب تطبيقها باستخدام البنى المادية (مصفوفة أو قائمة مرتبطة)
  • العمليات الأساسية: push()، pop()، top()، isEmpty()، isFull()
  • جميع العمليات O(1) - زمن ثابت
  • يمكن تطبيقه باستخدام ArrayList (مصفوفة ثابتة/ديناميكية) أو LinkedList
  • التطبيقات لا محدودة وتعتمد على المشكلة
  • تطبيق مهم: تقييم التعابير (تحويل Infix إلى Postfix)
  • يُفضّل الحاسب ترميز Postfix لسهولة تقييمه بالمكدسات

لماذا ندرس المكدسات؟

المكدسات أساسية في علوم الحاسب وتظهر في كل مكان:

  • كل برنامج تُشغّله يستخدم مكدس الاستدعاء للدوال
  • متصفح الويب الخاص بك يستخدم مكدسات للتنقل للخلف/للأمام
  • المُترجمات (Compilers) تستخدم مكدسات لتحليل التعابير وتقييمها
  • وظيفة التراجع/الإعادة في التطبيقات تعتمد على المكدسات
  • كثير من الخوارزميات (DFS، التراجع) تعتمد على سلوك المكدس

إن فهم المكدسات ضروري لتصبح مبرمجاً محترفاً!
