Course Introduction

Welcome to CYB 236: Web Application Security

What is Web Application Security?

Web Application Security (WAS) is the practice of protecting websites, web applications, and web services from security threats that exploit vulnerabilities in application code, design, or configuration. This course focuses on understanding web application concepts and techniques for managing, controlling, and improving security.

Why Web Security Matters

Web applications handle sensitive data including personal information, financial records, and health data. Attacks can lead to data breaches, financial loss, and reputation damage. The average cost of a data breach in 2024 exceeded $4.45 million. Regulatory compliance (GDPR, PCI-DSS) requires robust security measures.

The Origins of Hacking

Understanding the historical roots of security

Before delving into actual offensive and defensive security techniques, it is important to have at least some understanding of software security's long and interesting history. A brief overview of major security events in the last one hundred years should give you an understanding of the foundational technology underlying today's web applications.

In the past two decades, hackers have gained more publicity and notoriety than ever before. It's easy to assume that hacking is a concept closely tied to the internet and that most hackers emerged in the last 20 years. But that's only a partial truth.

Key Insight

While the number of hackers worldwide has exploded with the rise of the World Wide Web, hackers have been around since the middle of the 20th century - potentially even earlier depending on how you define "hacking."

Types of Hackers

The term "hacking" has evolved significantly over time. Originally, it referred to clever technical solutions - finding elegant ways to solve complex problems. Today, the security community distinguishes between different types of hackers:

White Hat Hackers
Ethical hackers who find vulnerabilities to improve security. They work with permission and follow responsible disclosure practices.
Black Hat Hackers
Malicious actors who exploit vulnerabilities for personal gain, theft, or damage without authorization.
Grey Hat Hackers
Operate in the middle - may find vulnerabilities without permission but typically don't have malicious intent.
Security Researchers
Professionals who study systems to identify and help fix security weaknesses through systematic analysis.

To understand how to defend systems, we must first understand how they can be attacked.

- Fundamental principle of security education

Historical Timeline

Major events that shaped cybersecurity

Circa 1930

The Enigma Machine

The Enigma machine used electric-powered mechanical rotors to both encrypt and decrypt text-based messages sent over radio waves. The device had German origins and would become an important technological development during the Second World War.

The device looked like a large mechanical typewriter. On each key press, the rotors would move and record a seemingly random character that would then be transmitted to all nearby Enigma machines. However, these characters were not random - they were defined by the rotation of the rotor and configuration options that could be modified.

How It Worked

  • Symmetric Key Encryption: Any Enigma machine with a specific configuration could read or "decrypt" messages sent from another machine with an identical configuration
  • Configuration Complexity: The rotors created billions of possible configurations, making manual decryption nearly impossible
  • Military Adoption: The German military adopted it in 1928 to secure top-secret military messages in transit
Security Lesson

The Enigma represents the first major "security system" that would later be compromised - demonstrating that even the most complex systems can have vulnerabilities. The type of encryption used is now known as a symmetric key algorithm, still used today in software to secure data in transit.

The First "Hackers"

In September 1932, Polish mathematician Marian Rejewski was provided a stolen Enigma machine. Along with colleagues Jerzy Rozycki and Henryk Zygalski, he reverse-engineered the system by analyzing patterns in encrypted messages and understanding the mechanics of the machine. By 1933, the team was intercepting and decrypting Enigma traffic on a daily basis.

Much like hackers today, Marian and his team intercepted and reverse-engineered encryption schemes to get access to valuable data. For these reasons, they can be considered some of the world's earliest hackers.

Circa 1940

Alan Turing & The Bombe

Alan Turing was an English mathematician often called the father of computer science. Prior to and during World War II, his research focused primarily on cryptography. Starting in September 1938, he worked at the Government Code and Cypher School (GC&CS) at Bletchley Park, England.

The Polish methods had become ineffective because the Germans had increased the number of rotors in their Enigma machines. Turing needed a different type of solution - one that would scale and could be used to break new types of encryption.

The Bombe Machine

A bombe was an electric-powered mechanical device that attempted to automatically reverse-engineer the position of mechanical rotors in an Enigma machine based on analysis of intercepted messages.

  • Key Insight: Turing realized that the German military sent daily weather reports in a predictable format - this "known plaintext" made cracking the encryption much easier
  • Automation: The machine had 108 drums rotating at up to 120 RPM, testing nearly 20,000 possible configurations in just 20 minutes
  • Impact: Breaking Enigma is estimated to have shortened WWII by 2-4 years, saving millions of lives
Technical Term: Known Plaintext Attack (KPA)

Today, Turing's reverse-engineering strategy is known as a Known Plaintext Attack (KPA) - an algorithm made much more efficient by being provided with prior input/output data. Similar techniques are used by modern hackers to break encryption. The Bombe marked an important point in history as one of the first automated hacking tools ever built.

Circa 1950-1960

Telephone Phreaking

In the late 1950s, telecoms like AT&T began implementing phones that could be automatically routed to a destination number based on audio signals. Pressing a key emitted a specific audio frequency that was transmitted over the line and interpreted by a switching machine. This was called tone dialing.

Within a short period, small groups realized that systems built on audio tone interpretation could be easily manipulated. Simply reproducing identical audio frequencies could interfere with the intended functionality. These hobbyists became known as phreakers - an early type of hacker specializing in manipulating telephone networks.

The 2600 Hz Tone

The most notorious case was the discovery of the 2600 Hz tone - an internal AT&T signal indicating a call had ended. Emitting this tone could prevent billing systems from recording calls, allowing free international calls.

  • Joe Engressia: A young boy with perfect pitch who could whistle at 2600 Hz and discovered he could manipulate phone systems
  • John Draper (Captain Crunch): Discovered that toy whistles in Cap'n Crunch cereal boxes produced a 2600 Hz tone
  • Blue Boxes: Hardware devices that generated specific frequencies, allowing anyone to exploit the free calling bug
Famous Phreakers

Steve Wozniak and Steve Jobs (Apple founders) built and sold blue boxes before starting Apple Computer. This demonstrates how the hacker mindset of understanding and manipulating systems has driven technological innovation.

The Response: DTMF

In the 1960s, phones were equipped with Dual-Tone Multi-Frequency (DTMF) signaling, commonly known as "Touch Tones." Each key emitted two very specific frequencies simultaneously, making replication much harder. This was the first major security development introduced specifically to combat phreakers.

Critical Lesson

Technology designed without considering users with malicious intent will eventually be broken into. Always consider the worst-case scenario when designing systems deployed at scale. This lesson remains essential for modern web application security.

Circa 1980

The Computer Hacking Era

In 1976, Apple released the Apple 1 personal computer. In 1982, the Commodore 64 was released - a fully configured personal computer right out of the box that would sell nearly 500,000 units per month until the early 1990s. Computers became common tools in households and businesses.

The First Computer Virus (1983)

Fred Cohen, an American computer scientist, created the very first computer virus. The virus could make copies of itself and spread from computer to computer via floppy disk. He stored it inside a legitimate program, masking it from detection. Cohen later demonstrated that detecting viruses from valid software with algorithms was almost impossible.

The Morris Worm (1988)

Robert Morris deployed the first virus that infected computers outside a research lab. The "Morris Worm" was a self-replicating virus that spread to about 15,000 network-attached computers within the first day.

  • Damage: The US Government Accountability Office estimated damage at $10,000,000
  • Consequence: Robert Morris became the first convicted hacker in the United States
  • Sentence: Three years probation, 400 hours community service, and a $10,050 fine
  • Legacy: Led to creation of CERT (Computer Emergency Response Team)
Legislation Response

The Computer Fraud and Abuse Act (CFAA) was enacted in 1986, establishing federal crimes related to computer intrusion. This was the first major US legislation specifically addressing computer crimes.

Modern techniques still mirror those from the 1980s: scalability (jumping from one user to another) and camouflaging (hiding malicious code inside legitimate programs) remain core techniques, now targeting web browsers instead of operating systems.

Circa 2000

Rise of the World Wide Web

The World Wide Web emerged in the 1990s, but its popularity exploded at the end of the 1990s and early 2000s. Initially, the web was almost exclusively used for sharing HTML documents. Websites didn't pay attention to user experience, and very few allowed users to send inputs back to the server.

The Web 2.0 Revolution

The early 2000s marked a new era: Web 2.0. Websites began to store user-submitted data and modify functionality based on user input. Users could collaborate by submitting data over HTTP to web servers, which would store and share it with other users.

This ideology gave birth to social media, blogs, wikis, and media sharing sites. The web transformed from a document-sharing platform to an application distribution platform.

New Attack Vectors Emerge

This shift in architecture changed how hackers targeted web applications. With serious efforts taken to secure servers and networks, the user became the perfect target.

  • SQL Injection: Discovered in the late 1990s, became one of the most common web attacks
  • Cross-Site Scripting (XSS): First documented in 2000, allowed attackers to inject malicious scripts into legitimate websites
  • Denial of Service (DoS): Early 2000s attacks shut down Yahoo!, Amazon, eBay, and other popular sites
  • Phishing: By mid-2000s, credential-stealing websites were rampant with no controls to protect users
CardSystems (2005)
40 million credit cards exposed in one of the first major web-based data breaches.
TJX Companies (2007)
94 million customer records stolen through SQL injection vulnerabilities.
OWASP Founded (2001)
The Open Web Application Security Project began providing free security resources to developers.
Circa 2015+

Hackers in the Modern Era

Today's web applications are much larger and more complex than their predecessors. A well-known web application can have hundreds of open source dependencies, integrations with other websites, multiple databases, and be served from multiple servers in multiple locations.

Modern Threat Landscape

  • Advanced Persistent Threats (APTs): Long-term, targeted attacks often sponsored by nation-states
  • Ransomware: Attacks that encrypt data and demand payment (WannaCry, NotPetya)
  • Supply Chain Attacks: Compromising software vendors to attack their customers (SolarWinds 2020)
  • Bug Bounty Programs: Companies paying ethical hackers to find vulnerabilities (Google, Microsoft, Apple)

Major Modern Breaches

Equifax (2017)
147 million records exposed due to an unpatched Apache Struts vulnerability - a simple fix that wasn't applied.
Capital One (2019)
100 million records stolen through a misconfigured AWS firewall - showing the importance of cloud security.
Log4j (2021)
Critical vulnerability in a widely-used Java logging library affected millions of systems worldwide.
SolarWinds (2020)
Supply chain attack compromised thousands of organizations including US government agencies.

Modern Browser Security

Browsers have become significantly more advanced with new security features:

  • Same Origin Policy (SOP): Robust isolation between websites with different origins
  • Content Security Policy (CSP): Allows developers to specify security levels for their applications
  • HTTPS/TLS: Encryption for data in transit, making man-in-the-middle attacks very difficult
Key Takeaway

As a result of browser security improvements, the most successful hackers today target application logic and code vulnerabilities rather than the browser itself. Modern hackers spend most of their time exploiting bugs in web application code, not trying to break the underlying platform.

Essential concepts from this topic

Cat and Mouse

Key Terms to Remember

Enigma Machine
WWII-era encryption device using symmetric key cryptography that demonstrated no system is unbreakable.
Known Plaintext Attack (KPA)
Cryptographic attack using known input/output pairs to deduce encryption keys or algorithms.
Phreaking
Early form of hacking targeting telephone systems using audio frequency manipulation.
Morris Worm
First major Internet worm (1988) that led to creation of CERT and the first hacker conviction.
Web 2.0
The shift from static document sharing to interactive, user-driven web applications.
APT (Advanced Persistent Threat)
Sophisticated, long-term targeted attacks, often nation-state sponsored.

Each new technology comes with its own unique attack surface and vulnerabilities. One way to become an excellent security professional is to always stay up to date on the latest technologies - these will often have security holes not yet published or found.

- Andrew Hoffman, Web Application Security
