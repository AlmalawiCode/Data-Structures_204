📚 Why the Orders Are Different

Pre-order: D B A C F E G

The root D is read first, then we go deep into the left side (B, A, C) before touching the right side (F, E, G). Useful for: copying a tree, prefix expression.

In-order: A B C D E F G

We go all the way left first. A (deepest left) is read first, then back to B, then C, then D (root!), then right side. Useful for: BST gives sorted order.

Post-order: A C B E G F D

All children must be finished before reading the parent. D (root) is read last. Useful for: deleting a tree, postfix expression.

🌐 Breadth-First Search (Level-Order Traversal)
Instead of going deep, BFS visits nodes level by level, left to right. It uses a Queue data structure: enqueue the root, then repeatedly dequeue a node, visit it, and enqueue its children.

👉 BFS Algorithm (uses a Queue)

1. Enqueue the root
2. While queue is not empty:
   a. Dequeue front node → VISIT it ✓
   b. Enqueue its left child (if exists)
   c. Enqueue its right child (if exists)

Why is BFS different from DFS? BFS explores the tree wide before going deep. It uses a Queue (FIFO) while DFS uses a Stack (implicitly through recursion). BFS guarantees visiting nodes in level order: all of level 0, then all of level 1, etc.
