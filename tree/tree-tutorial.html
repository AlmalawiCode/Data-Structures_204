🔑 Key Difference

All previous data structures (Array, Linked List, Stack, Queue) are LINEAR — you move through them one step at a time in a single direction. The Tree is completely different: it is NON-LINEAR and HIERARCHICAL. This makes it powerful for representing real-world relationships.

The Tree is one of the most important data structures in all of computer science. Operating system file systems, search algorithms, compilers, databases, and AI decision trees are all built using tree structures. Let us explore why!

3Linear vs Non-Linear Data Structures

This is the most fundamental concept to understand. What does it mean for a data structure to be linear or non-linear?

■ Array — Linear, Consecutive Memory

Elements are stored in consecutive memory locations. One comes directly after the other. Movement is one-directional: forward.

🔗 Linked List — Linear, Scattered Memory

Nodes are stored at random (scattered) memory addresses. However, each node stores the address of the next node (a pointer), so we traverse them in a linear, sequential order. It is scattered in memory, but the logical order is still linear.

Array & Linked List: Both are linear. Each element/node has exactly one predecessor and one successor. You can only move in one direction.

🌳 Tree — Non-Linear, Hierarchical

In a tree, a single node can connect to multiple children. From any node you can go down-left or down-right (or more). There is no single "next" element. This branching is what makes the tree non-linear.

6Measuring a Tree — Height & Depth

How do we measure the "size" of a tree? We do not use length like an array or linked list. Instead, we use height and depth.

📈 Height of a Tree

The height of a tree is the number of edges on the longest path from the root to any leaf.

💡 Count the edges, not the nodes!

📈 Depth of a Node

The depth of a node is the number of edges from the root to that node.

The depth of the root is always 0.

🔑 Rules for Height

  • A tree with only the root (single node) has height = 0
  • Height = number of edges on the longest root-to-leaf path
  • Height = (number of nodes on longest path) − 1
  • The depth of the root is always 0
  • The height of any leaf node is 0

7Why the Tree is a Logical Data Structure

We say the tree is a logical data structure. This means its shape is defined by rules and constraints, not by how data happens to be arranged in physical memory. The most important rule is: a tree must have no cycles.

🔑 Tree Rules — What makes it a TREE?

  • There is exactly one root node
  • Every node (except root) has exactly one parent
  • There are no cycles (no path from a node back to itself)
  • The tree is connected (there is a path between any two nodes)

If any of these rules is broken, it is NOT a tree — it becomes a general graph.

9Binary Tree — The Heart of CS Trees

📚 Definition

A Binary Tree is a tree data structure in which each node has at most two children, referred to as the left child and the right child. There is exactly one root. Each child is itself a binary tree (or empty).

💬 Properties of a Binary Tree

  • Each node has: a data value, a reference to the left child, and a reference to the right child
  • Either child can be null (empty) — nodes can have 0, 1, or 2 children
  • The left and right positions are distinct — a left child is different from a right child even if values are the same

💻 Node Structure in Java

class Node {
    int data;            // the value stored
    Node left;           // reference to left child
    Node right;          // reference to right child

    Node(int val) {
        data = val;
        left = null;      // no left child initially
        right = null;     // no right child initially
    }
}

10Tree Levels & Maximum Nodes

In a binary tree, nodes are organized by levels. The root is at level 0, its children at level 1, their children at level 2, and so on.

Formula: The maximum number of nodes at level i is 2i
  • Level 0 (root): max 20 = 1 node
  • Level 1: max 21 = 2 nodes
  • Level 2: max 22 = 4 nodes
  • Level 3: max 23 = 8 nodes
  • Level i: max 2i nodes

11Subtrees

A subtree is a smaller tree formed by taking any node in the tree and all of its descendants. Every node in a tree is the root of its own subtree!

📚 Definition

A subtree rooted at node X consists of node X and all its descendants (children, grandchildren, etc.). By definition, a subtree is itself a valid binary tree.

13Complete Binary Tree

📚 Definition

A Complete Binary Tree is a binary tree in which:
1. All levels are completely filled except possibly the last level.
2. The last level has all nodes as far left as possible.

14Tree Traversal

How do we visit every node in a tree? In an array or linked list, we go one by one in order. In a tree, there are multiple ways to traverse. We call these traversal algorithms.

📲 Depth-First Search (DFS)

Go as deep as possible before backtracking. Three variants:

  • Pre-order (Root, Left, Right)
  • In-order (Left, Root, Right)
  • Post-order (Left, Right, Root)

🌐 Breadth-First Search (BFS)

Visit all nodes at each level before going deeper. Also called Level-Order traversal.

Uses a Queue to process nodes level by level.

💡 How to Remember: The "3 Visits" Idea

Imagine walking through the tree. Each time you pass through a node, count the visit:

  • 1st visit (arrive) → Pre-order — visit the root when you arrive at it
  • 2nd visit (return from left) → In-order — visit when you come back from the left subtree
  • 3rd visit (return from right) → Post-order — visit when you come back from the right subtree

Pre-order

Root → Left → Right

First visit the root, then recursively traverse the left subtree, then the right subtree.

Used for: copying a tree, prefix notation

In-order

Left → Root → Right

First traverse the left subtree, then visit the root, then the right subtree.

Used for: BST gives sorted order!

Post-order

Left → Right → Root

First traverse left, then right, then visit the root last.

Used for: deleting a tree, postfix notation

🌐 Breadth-First Search (Level-Order) — How it Works

BFS uses a Queue data structure. The algorithm:

  1. Start: Enqueue the root
  2. While the queue is not empty:
    1. Dequeue the front node and visit it
    2. Enqueue its left child (if it exists)
    3. Enqueue its right child (if it exists)

This guarantees we visit nodes level by level, left to right.

💻 Java Code for All Traversals

// Pre-order: Root → Left → Right
void preOrder(Node node) {
    if (node == null) return;
    System.out.print(node.data + " "); // visit ROOT first
    preOrder(node.left);
    preOrder(node.right);
}

// In-order: Left → Root → Right
void inOrder(Node node) {
    if (node == null) return;
    inOrder(node.left);
    System.out.print(node.data + " "); // visit ROOT in middle
    inOrder(node.right);
}

// Post-order: Left → Right → Root
void postOrder(Node node) {
    if (node == null) return;
    postOrder(node.left);
    postOrder(node.right);
    System.out.print(node.data + " "); // visit ROOT last
}

// Level-order (BFS): using a Queue
void levelOrder(Node root) {
    if (root == null) return;
    Queue<Node> queue = new LinkedList<>();
    queue.add(root);
    while (!queue.isEmpty()) {
        Node n = queue.poll();
        System.out.print(n.data + " ");
        if (n.left != null) queue.add(n.left);
        if (n.right != null) queue.add(n.right);
    }
}
