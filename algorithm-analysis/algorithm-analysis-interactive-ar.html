①
رمز O الكبير (Big O Notation) كيفية قراءة أي كود جافا، وحساب عملياته، والتعبير عن النتيجة بصيغة O الكبير.
②
زمن التنفيذ (Execution Time) ما معنى زمن التنفيذ، وكيفية استخدام المعادلة T(n) = c × g(n)، وكيفية توقع أوقات التشغيل.
كيفية استخدام هذه الصفحة: يحتوي كل قسم على أمثلة تعليمية واضحة — ادرسها أولاً، ثم حاول حل مسائل التدريب بنفسك. اضغط على أظهر الحل للتحقق من إجابتك.

① رمز O الكبير (Big O Notation)

يصف رمز O الكبير مدى نمو الخوارزمية مع زيادة حجم المدخلات n. لإيجاد رمز O الكبير لأي كود جافا، اتبع خطوتين:

الخطوة 1 — كود جافا ← كثيرات الحدود
1
جملة بسيطة (Simple Statement)كل جملة = 1 عملية
2
حلقة تكرار (Loop)اضرب عدد التكرارات × العمليات بداخلها
3
حلقات متداخلة (Nested Loops)اضرب أعداد تكرارات جميع الحلقات
4
كود متسلسل (Sequential Code)اجمع عمليات كل كتلة
أمثلة تعليمية
مثال 1 حلقة واحدة (Single Loop)
☕ كود جافا ▼
for (int i = 0; i < n; i++) { sum = sum + i; // 1 operation }
→

الحلقة تُنفَّذ: n مرة  ×  1 عملية داخلها = n

f(n) = n
مثال 2 حلقات متداخلة (n × n)
☕ كود جافا▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { matrix[i][j] = 0; // 1 operation } }
→

الحلقة الخارجية: n مرة  ×  الحلقة الداخلية: n مرة  ×  1 عملية = n²

f(n) = n²
مثال 3 حلقات متسلسلة (اجمعها)
☕ كود جافا▼
for (int i = 0; i < n; i++) { sum += i; // 1 operation } for (int j = 0; j < n*n; j++) { delta += 1; // 1 operation }
1

الحلقة الأولى: تُنفَّذ n مرة → n عملية

2

الحلقة الثانية: تُنفَّذ n² مرة → n² عملية

+

متسلسلة → اجمع: T(n) = n + n²

f(n) = n² + n
مثال 4 حلقة لوغاريتمية (التنصيف)
☕ كود جافا▼
while (n > 0) { System.out.println(n); n = n / 2; }
1

بعد كل تكرار يتنصف n: n → n/2 → n/4 → n/8 → … → 1

2

بعد k تكرار: n / 2ᵏ = 1 → 2ᵏ = n → k = log₂(n)

f(n) = log₂(n)
مثال 5 حلقة داخلية مثلثية (j < i)
☕ كود جافا▼
for (int i = 0; i < n; i++) { for (int j = 0; j < i; j++) { sum += i * j; // 1 operation } }
1

الحلقة الداخلية تُنفَّذ: 0، 1، 2، 3، …، n−1 مرة لكل قيمة i

2

المجموع = 0 + 1 + 2 + … + (n−1) = n(n−1)/2

3

التوسيع: n(n−1)/2 = (n²−n)/2 = n²/2 − n/2

f(n) = (1/2)n² − (1/2)n
الخطوة 2 — كثيرات الحدود ← O الكبير
1
احتفظ بالحد الأعلى درجةالحد ذو الأس الأكبر
2
احذف جميع المعاملات76,000,000n² → n²
3
احذف الحدود الأدنى درجةn² + n + 5 → n²
4
الثابت → O(1)f(n) = 42 → O(1)
لماذا؟ عندما يتجه n نحو اللانهاية، يسيطر الحد الأعلى درجة كلياً. مثلاً: لـ f(n) = 4n² + 3n + 10: عند n = 1,000، الجزء 4n² = 4,000,000 يتفوق بشكل كبير على 3n (وهو 3,000) وعلى 10.
أمثلة تعليمية
م1 كثيرة حدود تربيعية

f(n) = 4n² + 3n + 10

1

الحد الأعلى: 4n²

2

احذف المعامل 4 → n²

O(n²)

f(n) = 76,756,234n² + 427,913n + 7

1

الحد الأعلى: 76,756,234n²

2

احذف المعامل → n²

O(n²)
م3 كثيرة حدود عالية الدرجة

f(n) = 74n⁸ − 62n⁵ − 71,562n³ + 3n² − 5

1

الحد الأعلى: 74n⁸

2

احذف المعامل → n⁸

O(n⁸)
م4 وسّع أولاً

2

احذف المعامل → n¹⁰

O(n¹⁰)
م5 n log n

1

الحد الأعلى: 75n log n

2

احذف 75 → n log n

O(n log n)

تدريب 1 حلقة عداد بسيطة
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
int count = 0; for (int i = 0; i < n; i++) { count++; // how many operations? }

✅ الحل

1

الحلقة تبدأ من i = 0 وتستمر حتى i < n — أي ما مجموعه n تكراراً بالضبط.

2

داخل الحلقة، تُنفَّذ عملية واحدة فقط (count++) في كل تكرار.

3

إجمالي العمليات = n × 1 = n

4

لا توجد ثوابت لحذفها — الحدّ السائد هو n مباشرةً.

f(n) = n  O(n)
تدريب 2 حلقة متداخلة (j تبدأ من i)
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
for (int i = 0; i < n; i++) { for (int j = i; j < n; j++) { sum += array[i][j]; } }

✅ الحل

1

الحلقة الخارجية تُنفَّذ للقيم i = 0، 1، 2، …، n−1.

2

الحلقة الداخلية تبدأ من j = i، لذا يتغير عدد تكراراتها مع كل قيمة لـ i:
  i = 0 → الداخلية تُنفَّذ n مرة
  i = 1 → الداخلية تُنفَّذ n−1 مرة
  i = 2 → الداخلية تُنفَّذ n−2 مرة
  ⋮
  i = n−1 → الداخلية تُنفَّذ مرة واحدة

3

المجموع = n + (n−1) + (n−2) + … + 1 = n(n+1)/2

4

نوسّع الصيغة → n²/2 + n/2

5

الحدّ السائد هو n²/2 — نحذف المعامل الثابت ½ والحدّ الأصغر n/2.

f(n) = n²/2 + n/2  O(n²)
تدريب 3 ثلاث حلقات متداخلة
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { for (int k = 0; k < n; k++) { cube[i][j][k] = i + j + k; } } }

✅ الحل

1

الحلقة الخارجية تعمل من i = 0 حتى i < n → n تكرار.

2

لكل قيمة من i، الحلقة الوسطى تعمل من j = 0 حتى j < n → n تكرار إضافي.

3

لكل زوج (i، j)، الحلقة الداخلية تعمل من k = 0 حتى k < n → n تكرار إضافي.

4

إجمالي العمليات = n × n × n = n³

5

لا يوجد معامل ثابت — الحدّ السائد هو n³.

f(n) = n³  O(n³)
تدريب 4 حلقة تتضاعف في كل خطوة
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
int i = 1; while (i < n) { System.out.println(i); i = i * 2; }

✅ الحل

1

تبدأ i بالقيمة 1 وتتضاعف في كل تكرار: 1، 2، 4، 8، …، 2ᵏ

2

تتوقف الحلقة عند i ≥ n، أي نحتاج أن يحقق 2ᵏ ≥ n.

3

نحلّ للحصول على k: نضع 2ᵏ = n ثم نطبّق log₂ على الطرفين → k = log₂(n).

4

أساس اللوغاريتم يختلف فقط بمعامل ثابت — جميع الأسس متكافئة في O الكبير.

f(n) = log₂(n)  O(log n)
تدريب 5 مختلطة: حلقة متداخلة + حلقة واحدة
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { matrix[i][j] = 0; } } for (int k = 0; k < n; k++) { sum += array[k]; }

✅ الحل

1

القسم الأول — الحلقة المتداخلة: الخارجية n تكرار، الداخلية n تكرار → n × n = n² عملية.

2

القسم الثاني — الحلقة الواحدة: تعمل من k=0 حتى k<n → n عملية.

3

القسمان يعملان بشكل متسلسل (أحدهما بعد الآخر) — نجمع تكاليفهما: n² + n.

4

الحدّ السائد هو n² (ينمو أسرع بكثير من n). نحذف الحدّ الأصغر n.

f(n) = n² + n  O(n²)
تدريب 6 بحث مع خروج مبكر
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
boolean found = false; for (int i = 0; i < n && !found; i++) { if (array[i] == target) { found = true; } }

✅ الحل

1

الحلقة تعمل من i=0 حتى i<n، لكنها قد تتوقف مبكراً إذا وُجد الهدف (found = true).

2

أفضل حالة: الهدف موجود في الفهرس 0 → عملية واحدة فقط.

3

أسوأ حالة: الهدف غير موجود في المصفوفة → تُنفَّذ الحلقة كاملاً n مرة.

4

O الكبير يقيس دائماً أسوأ الحالات — لذا نستخدم n.

f(n) = n  O(n)
تدريب 7 نمط النجوم المثلثي
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { System.out.print("*"); } System.out.println(); }

✅ الحل

1

الحلقة الخارجية تعمل للقيم i = 1، 2، …، n.

2

الحلقة الداخلية تعمل من j = 1 حتى j ≤ i، لذا يزداد عدد تكراراتها مع i:
  i = 1 → الداخلية تُنفَّذ مرة واحدة
  i = 2 → الداخلية تُنفَّذ مرتين
  i = 3 → الداخلية تُنفَّذ 3 مرات
  ⋮
  i = n → الداخلية تُنفَّذ n مرة

3

المجموع = 1 + 2 + 3 + … + n = n(n+1)/2

4

نوسّع → n²/2 + n/2. الحدّ السائد هو n²/2 — نحذف المعامل ½ والحدّ الأصغر n/2.

f(n) = n²/2 + n/2  O(n²)
تدريب 8 عمليات ذات زمن ثابت
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.
☕ كود جافا▼
int first = array[0]; int last = array[n - 1]; int mid = array[n / 2]; int sum = first + last + mid;

✅ الحل

1

array[0] — عملية واحدة: وصول مباشر بالفهرس، لا حاجة لحلقة.

2

array[n−1] — عملية واحدة: وصول مباشر بالفهرس بغض النظر عن قيمة n.

3

array[n/2] — عملية واحدة: وصول مباشر بالفهرس بغض النظر عن قيمة n.

4

first + last + mid — عملية واحدة: جمع بسيط.

5

الإجمالي = 4 عمليات — مستقل تماماً عن n. مهما كان حجم المصفوفة، دائماً 4 عمليات بالضبط.

6

أي ثابت ثابت يُبسَّط إلى O(1).

f(n) = 4  O(1)
تدريب 9 حلقة تنازلية

✅ الحل

1

الحلقة تبدأ من i = n وتتناقص: n، n−1، n−2، …، 1.

2

العدّ التنازلي من n إلى 1 يعطي بالضبط n تكراراً — نفس عدد العدّ التصاعدي.

إجمالي العمليات = n × 1 = n.

4

اتجاه العدّ (تصاعدي أو تنازلي) لا يؤثر على O الكبير — المهم هو عدد التكرارات.

f(n) = n  O(n)

تدريب 11 متغيران مختلفان (n، m)
حلّل كود جافا التالي. أوجد كثيرات الحدود f(n) وحدّد رمز O الكبير لها.

✅ الحل

!

f(n,m) = n × m  O(n·m)

1000n + 500 → الحد الأعلى 1000n، احذف المعامل →  O(n)

3

n² log n + 3n² → n² log n يسيطر على n² →  O(n² log n)

4

2ⁿ + n³ → الأسي يتفوق على كثيرات الحدود →  O(2ⁿ)

5

f(n) = 15 → ثابت، لا يوجد n →  O(1)

② زمن التنفيذ — توقع أوقات التشغيل

بمجرد معرفة رمز O الكبير للخوارزمية، يمكننا توقع مدة تشغيلها لأي حجم مدخلات. نستخدم المعادلة:

المعادلة

إذا كانت الخوارزمية من النوع O(g(n))، فإن زمن تشغيلها يُنمذج كالتالي:

T(n) = c × g(n)

حيث c ثابت يعتمد على العتاد والتنفيذ. استخدم نقطة بيانات معروفة (قيمة n والزمن المعطيان) لإيجاد c، ثم استخدم c للتنبؤ بأي زمن آخر.

1
إيجاد الثابت cعوّض بالزوج المعطى (n، الزمن)
2
استخدام cعوّض بقيمة n المستهدفة للحصول على T(n)
3
إيجاد nعوّض بالزمن المستهدف وأوجد n
أمثلة تعليمية
مثال 1 O(n) — أوجد الزمن لقيمة n جديدة
الخوارزمية من النوع O(n). عند n = 10 الزمن = 2 ملي ثانية. أوجد الزمن عند n = 500.
1
اكتب معادلة الزمن
T(n) = c × n
2
عوّض بالقيم المعروفة  (n = 10, T = 2 ms)
T(10) = c × 10
2 = c × 10
3
أوجد قيمة c
c = 2 ÷ 10
c = 1/5
4
أوجد الزمن عند n = 500
T(500) = c × n
T(500) = (1/5) × 500
T(500) = 100 ms
✅ الجواب: 100 ملي ثانية
مثال 2 O(n²) — أوجد الزمن لقيمة n جديدة
الخوارزمية من النوع O(n²). عند n = 4 الزمن = 10 ملي ثانية. أوجد الزمن عند n = 16.
1
اكتب معادلة الزمن
T(n) = c × n²
2
عوّض بالقيم المعروفة  (n = 4, T = 10 ms)
T(4) = c × 4²
10 = c × 16
3
أوجد قيمة c
c = 10 ÷ 16
c = 5/8
4
أوجد الزمن عند n = 16
T(16) = c × n²
T(16) = (5/8) × 16²
T(16) = (5/8) × 256
T(16) = 160 ms
✅ الجواب: 160 ملي ثانية
مثال 3 O(log₂ n) — أوجد الزمن لقيمة n جديدة
الخوارزمية من النوع O(log₂ n). عند n = 16 الزمن = 28 ملي ثانية. أوجد الزمن عند n = 64.

مثال 4 O(n²) — أوجد حجم المدخلات n
الخوارزمية من النوع O(n²). عند n = 10 الزمن = 30 ملي ثانية. أوجد n عندما يكون الزمن = 120 ملي ثانية.
1

2

3

c = 3/10
4
عوّض بالزمن المطلوب (T = 120 ms) وأوجد n
120 = (3/10) × n²
n² = 120 × (10/3)
n² = 1200 ÷ 3
n² = 400
n = √400
n = 20
✅ الجواب: n = 20

🔥 مسائل التحديات

تحدٍّ 1 حلقة خارجية لوغاريتمية × حلقة داخلية خطية
☕ كود جافا▼
for (int i = 1; i < n; i = i * 2) { for (int j = 0; j < n; j++) { process(i, j); } }

✅ الحل

1

الحلقة الخارجية: i تتضاعف → تُنفَّذ log₂(n) مرة

2

الحلقة الداخلية: تُنفَّذ n مرة لكل تكرار خارجي

×

المجموع = n × log₂(n)

f(n) = n·log₂(n)  O(n log n)
تحدٍّ 2 مسألة خادعة — j لا تُعاد تهيئته
☕ كود جافا▼
int i = 0, j = 0; while (i < n) { while (j < n && array[i][j] == 1) { j++; } i++; }

✅ الحل — انتبه، تبدو O(n²) لكنها ليست كذلك!

!

j لا تُعاد تهيئتها عند زيادة i — تحتفظ بقيمتها بين تكرارات الحلقة الخارجية

1

i تزداد n مرة إجمالاً

2

j تزداد على الأكثر n مرة إجمالاً (لا تعود للخلف أبداً)

3

أقصى إجمالي للعمليات = 2n

f(n) = 2n  O(n)
تحدٍّ 3 دالة تعاودية (Recursive)
☕ كود جافا▼
void mystery(int n) { if (n <= 1) return; for (int i = 0; i < n; i++) { System.out.println(i); } mystery(n / 2); mystery(n / 2); }

✅ الحل

1

العلاقة التعاودية: T(n) = 2T(n/2) + n

2

كل مستوى يُنجز n عملية إجمالاً، وعمق التعاود = log₂(n)

3

المجموع = n × log₂(n) (هذا هو نمط خوارزمية Merge Sort!)

O(n log n)
تحدٍّ 4 حلقة خارجية جذرية (Square Root)
☕ كود جافا▼
for (int i = 1; i * i <= n; i++) { for (int j = 0; j < n; j++) { process(i, j); } }

✅ الحل

1

الخارجية: تُنفَّذ طالما i² ≤ n → i ≤ √n → √n تكرار

2

الداخلية: n تكرار في كل مرة → المجموع = √n × n = n^(3/2)

f(n) = n^(3/2)  O(n√n)
تحدٍّ 5 ثلاثي مثلثي (Triple Triangular)
☕ كود جافا▼
for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { for (int k = 1; k <= j; k++) { sum++; } } }

✅ الحل

1

لكل i، الحلقتان الداخليتان تُعطيان مثلث(i) = i(i+1)/2

2

المجموع = Σᵢ₌₁ⁿ i(i+1)/2 = n(n+1)(n+2)/6 ≈ n³/6

f(n) ≈ n³/6  O(n³)
تحدٍّ 6 قارن بين خوارزميتين
الخوارزمية A: O(n log n)  |  n = 8 → 24 ms
الخوارزمية B: O(n²)  |  n = 10 → 50 ms
أيهما أسرع عند n = 100؟

✅ الحل

A

24 = c × 8 × log₂(8) = c × 24 → c = 1
T(100) = 1 × 100 × log₂(100) ≈ 100 × 6.64 = 664 ms

B

50 = c × 100 → c = 0.5
T(100) = 0.5 × 10,000 = 5,000 ms

🏆 الخوارزمية A تفوز! (664 ms مقابل 5,000 ms)
