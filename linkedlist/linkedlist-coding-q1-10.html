Q1Easy
Traverse and Print a Singly Linked List
Write a method void printList(Node head) that prints all elements of a singly linked list separated by -> , ending with null.
head 10next → 20next → 30next → null
Expected Output: 10 -> 20 -> 30 -> null

Solution:

printList() Method
public void printList(Node head) {
    Node current = head;
    while (current != null) {
        System.out.print(current.data + " -> ");
        current = current.next;
    }
    System.out.println("null");
}
Explanation: We use a current pointer initialized to head. In each iteration of the while loop, we print the current node's data followed by " -> ", then advance the pointer with current = current.next. When current becomes null, we exit the loop and print "null" to indicate the end of the list.
Time Complexity: O(n) — we visit each node exactly once.
Space Complexity: O(1) — only one pointer variable is used.
Q2Easy
Count Nodes in a Linked List
Write a method int countNodes(Node head) that returns the total number of nodes in the list.
head 5next → 12next → 7next → 3next → null
Expected Output: 4

Solution:

countNodes() Method
public int countNodes(Node head) {
    int count = 0;
    Node current = head;
    while (current != null) {
        count++;
        current = current.next;
    }
    return count;
}
Explanation: Initialize a count variable to 0. Traverse the list using a current pointer, incrementing count at each node. When current reaches null, count holds the total number of nodes.
Time Complexity: O(n) — we traverse the entire list once.
Space Complexity: O(1) — only a counter and a pointer are used.
Q3Easy
Insert at the Beginning
Write a method Node insertAtHead(Node head, int data) that inserts a new node at the beginning of the list and returns the new head.
Before:
head 20next → 30next → null
After inserting 10:
head 10next → 20next → 30next → null

Solution:

insertAtHead() Method
public Node insertAtHead(Node head, int data) {
    Node newNode = new Node(data);
    newNode.next = head;
    return newNode;
}
Explanation: Create a new node with the given data. Set the new node's next pointer to the current head. Return the new node as the new head of the list. This works even if head is null (empty list).
Time Complexity: O(1) — no traversal needed.
Space Complexity: O(1) — only one new node is created.
Q4Easy
Insert at the End
Write a method Node insertAtEnd(Node head, int data) that inserts a new node at the end of the list and returns the head.
Before:
head 10next → 20next → null
After inserting 30:
head 10next → 20next → 30next → null

Solution:

insertAtEnd() Method
public Node insertAtEnd(Node head, int data) {
    Node newNode = new Node(data);
    if (head == null) return newNode;
    Node current = head;
    while (current.next != null) {
        current = current.next;
    }
    current.next = newNode;
    return head;
}
Explanation: Create a new node. If the list is empty (head == null), return the new node as the head. Otherwise, traverse to the last node (where current.next == null) and set its next pointer to the new node.
Time Complexity: O(n) — we must traverse to the end of the list.
Space Complexity: O(1) — only one new node and one pointer are used.
Q5Easy
Search for a Value
Write a method boolean search(Node head, int target) that returns true if the target value exists in the list, false otherwise.
head (searching for 15) 4next → 8next → 15next → 23next → null
Search for 15 → returns true

Solution:

search() Method
public boolean search(Node head, int target) {
    Node current = head;
    while (current != null) {
        if (current.data == target) {
            return true;
        }
        current = current.next;
    }
    return false;
}
Explanation: Perform a linear search by traversing each node and comparing current.data with target. If a match is found, return true immediately. If we reach the end of the list without finding the target, return false.
Time Complexity: O(n) — worst case, we check every node.
Space Complexity: O(1) — only one pointer variable is used.
Q6Easy
Delete the First Node
Write a method Node deleteFirst(Node head) that removes the first node from the list and returns the new head.
Before:
head 10del → 20next → 30next → null
After:
head 20next → 30next → null

Solution:

deleteFirst() Method
public Node deleteFirst(Node head) {
    if (head == null) return null;
    return head.next;
}
Explanation: Simply return head.next, which becomes the new head. The old head node is no longer referenced and becomes eligible for garbage collection in Java. We handle the edge case of an empty list by checking if head is null.
Time Complexity: O(1) — no traversal needed.
Space Complexity: O(1) — no extra space used.
Q7Medium
Delete the Last Node
Write a method Node deleteLast(Node head) that removes the last node from the list and returns the head.
Before:
head 10next → 20next → 30del → null
After:
head 10next → 20next → null

Solution:

deleteLast() Method
public Node deleteLast(Node head) {
    if (head == null || head.next == null) {
        return null;
    }
    Node current = head;
    while (current.next.next != null) {
        current = current.next;
    }
    current.next = null;
    return head;
}
Explanation: We need to reach the second-to-last node so we can set its next to null. We traverse using the condition current.next.next != null to stop one node before the last. Edge cases: if the list is empty or has only one node, return null.
Time Complexity: O(n) — we traverse nearly the entire list.
Space Complexity: O(1) — only one pointer variable is used.
Q8Medium
Delete a Node by Value
Write a method Node deleteByValue(Node head, int value) that deletes the first occurrence of the given value from the list.
Delete 20:
head 10next → 20del → 30next → null
After:
head 10next → 30next → null

Solution:

deleteByValue() Method
public Node deleteByValue(Node head, int value) {
    if (head == null) return null;
    // If the head node holds the value to delete
    if (head.data == value) return head.next;
    Node current = head;
    while (current.next != null) {
        if (current.next.data == value) {
            current.next = current.next.next;
            return head;
        }
        current = current.next;
    }
    return head; // value not found
}
Explanation: First, check if the head itself contains the value; if so, return head.next. Otherwise, traverse the list looking at current.next.data. When found, skip over the target node by setting current.next = current.next.next. If the value is not in the list, the original head is returned unchanged.
Time Complexity: O(n) — worst case, the value is at the end or not present.
Space Complexity: O(1) — only one pointer variable is used.
Q9Medium
Insert at a Given Position
Write a method Node insertAtPosition(Node head, int data, int position) that inserts a new node at the given position (0-indexed).
Before:
head 10next → 20next → 30next → null
After inserting 25 at position 2:
head 10next → 20next → 25new → 30next → null

Solution:

insertAtPosition() Method
public Node insertAtPosition(Node head, int data, int position) {
    Node newNode = new Node(data);
    // Insert at the beginning
    if (position == 0) {
        newNode.next = head;
        return newNode;
    }
    Node current = head;
    for (int i = 0; i < position - 1 && current != null; i++) {
        current = current.next;
    }
    if (current == null) return head; // position out of bounds
    newNode.next = current.next;
    current.next = newNode;
    return head;
}
Explanation: If position is 0, insert at the head. Otherwise, navigate to the node at index position - 1. Then insert the new node by pointing newNode.next to current.next, and updating current.next to point to newNode. We also handle the case where the position exceeds the list length.
Time Complexity: O(n) — we may traverse up to n nodes to reach the position.
Space Complexity: O(1) — only one new node and one pointer are used.
Q10Medium
Reverse a Linked List
Write a method Node reverse(Node head) that reverses the linked list in-place and returns the new head.
Before:
head 1next → 2next → 3next → 4next → null
After:
head 4next → 3next → 2next → 1next → null

Solution:

reverse() Method
public Node reverse(Node head) {
    Node prev = null;
    Node current = head;
    while (current != null) {
        Node nextTemp = current.next; // save next node
        current.next = prev;            // reverse the link
        prev = current;                 // move prev forward
        current = nextTemp;             // move current forward
    }
    return prev; // prev is the new head
}
Explanation: Use three pointers: prev (starts at null), current (starts at head), and nextTemp (to save the next node before we overwrite the link). In each iteration: (1) save current.next in nextTemp, (2) reverse the link by pointing current.next to prev, (3) advance prev and current one step forward. When current becomes null, prev points to the new head of the reversed list.
Time Complexity: O(n) — we visit each node exactly once.
Space Complexity: O(1) — only three pointer variables are used (in-place reversal).
