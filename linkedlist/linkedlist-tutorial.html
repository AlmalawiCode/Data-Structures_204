Understanding Why We Need Linked Lists and How They Work

Part 1: Why Do We Need Linked Lists?

Before we learn about linked lists, we must first understand the problem they solve. To do this, let's review what you already know: Arrays.

Let's Start with Arrays

You already know that an array stores elements in contiguous (consecutive) memory locations. Let's visualize an array of 5 integers:

Array of 5 Elements in Memory

Array:
10 1000
20 1004
30 1008
40 1012
50 1016

Notice: Each element is exactly 4 bytes apart (integer size). Memory addresses are consecutive!

Question: If we created this array with 5 elements, can we extend it to add more elements (like a 6th element)?
Answer: NO! An array has a fixed size. Once created, it cannot be extended.

But WHY is an Array Fixed?

The array is fixed because it MUST use contiguous memory locations. But why is contiguous memory so important?

The Benefit of Contiguous Memory

When elements are stored consecutively, the CPU can access any element instantly using this simple formula:

Element Address = Base Address + (Index × Element Size)

Example: To find element at index 3:

Address = 1000 + (3 × 4) = 1000 + 12 = 1012

The CPU only asks the Memory Management Unit (MMU) once for the base address, then uses simple math (offset) to find any element. This is called O(1) Random Access - incredibly fast!

The Extension Problem

Now let's understand why we cannot extend the array:

Memory is Shared by Many Programs

Our Array:
10 1000
20 1004
30 1008
40 1012
50 1016
?? 1020
?? 1024

The next memory locations (1020, 1024...) are OCCUPIED by another program!

Key Understanding

Memory (RAM) is not only for your program. Many programs run simultaneously and share the same memory. The locations right after your array might be used by:

  • Another program's data
  • The operating system
  • Another array in your own program

Since we cannot guarantee that the next locations are free, we cannot extend the array!

But what if we allocate a new block of memory that's NOT contiguous with our array?

Non-Contiguous Memory - The Problem

Original Array:
10 1000
20 1004
30 1008
40 1012
50 1016
New Block: ... gap in memory ...
60 2000

Problem: Address 2000 is NOT at offset 5 from 1000!
The formula 1000 + (5 × 4) = 1020 gives the WRONG address!
We lose the O(1) random access benefit!

Conclusion: If memory is not contiguous, we cannot use the offset formula. The CPU would need to search for where each element is stored, making access very slow. This defeats the main purpose of arrays!

Problems with This Approach

Problem 1: Very Slow!

If you have 1 million elements and want to add just ONE more, you must:

  1. Create a new array with 1,000,001 spaces
  2. Copy ALL 1 million elements one by one
  3. Add the new element

This is O(n) time complexity - very slow for large arrays!

Problem 2: Memory Fragmentation (The Critical Problem!)

This is the most important problem to understand. Let's see a real scenario:

Scenario: 2GB RAM Computer

Current Memory State:

Your Array
(1 GB)
FREE
(1 GB)
Your Array (1 GB)
Free Space (1 GB)
Question: You want to add just 1 KB of data to your 1 GB array. You have 1 GB free. Can you do it?
What You NEED vs What You HAVE

You need a NEW array of size: 1 GB + 1 KB

Needed: 1 GB + 1 KB
(Cannot create!)

IMPOSSIBLE! You only have 1 GB free, but need 1 GB + 1 KB!

Even though you have 1 GB FREE, you CANNOT add just 1 KB!

The Irony

You have 1 GB of free memory but cannot add 1 KB of data!

This is because you need to create a completely new array that's larger than the current one, and there's not enough contiguous space for it.

The "Disk Swap" Alternative

A student might suggest: "Why not save the old array to the hard disk, create the new larger array, then copy back?"

Yes, this is technically possible, but...

  • Hard disk read/write is 1000x slower than RAM
  • Writing 1 GB to disk takes significant time
  • Reading 1 GB back takes significant time
  • This would make a simple "add" operation take seconds or minutes!

This solution is impractical for most applications.

We Need a Better Solution!

These problems with arrays led developers to think: Is there a way to link data together without needing contiguous memory?

Let's explore this idea using something you already know from Object-Oriented Programming...

Part 4: Creating a Reusable Data Structure

What we built works great! But let me ask you something...
Question: What if tomorrow we need to link Car objects? Or Product objects? Or Employee objects? Should we add a nextCar variable to the Car class? A nextProduct to Product class?

Problems with Our Current Approach

  • We have to modify every class we want to link
  • Adding nextStudent, nextCar, etc. to every class is not good practice
  • The linking logic is mixed with the data class
  • We have to rewrite the traversal code for every type
Why not create our own reusable data structure that can hold ANY type of object? Let's call it... a Linked List!

The Linked List Concept

What is a Linked List?

A Linked List is a data structure that consists of:

  1. Nodes - containers that hold data and link to each other
  2. A Control Class - manages the nodes (add, remove, find, etc.)

The Node: A Universal Container

Instead of adding a "next" reference to every class (Student, Car, Product...), we create a separate container called a Node:

The Node Concept

A Node has two parts:

DATA
(any object)
NEXT
(reference)

DATA: Like a "car" or "box" - it carries any object (Student, Car, Integer, anything!)
NEXT: Points to the next Node in the chain

Think of it Like a Train

Each train car (Node) has:

  • A cargo area (DATA) - where you put whatever you want to transport
  • A coupler (NEXT) - connects to the next train car

The train cars don't care what's inside them - they just carry and connect!

The Control Class: LinkedList

We also need a control class that manages all the nodes. This class is responsible for:

Add
Add new nodes
Remove
Delete nodes
Insert
Insert at position
Search
Find elements
Traverse
Visit all nodes

Structure Overview

LinkedList (Control Class)
│
├── head (reference to first Node)
│
└── Node (Inner Class)
    ├── data (the actual object)
    └── next (reference to next Node)
Now let's build this step by step! We'll start simple and add more features as we go.

Part 5: Building Our LinkedList - Step by Step

Step 1: Create the Node Class (Generic)

First, let's create our Node class. We'll make it generic using <T> so it can store any type of object - Student, Car, Product, Integer, or anything else!

Node Class (Generic)
/**
 * Node class - A container that holds data and links to the next node.
 * Think of it as a "train car" that carries cargo and connects to the next car.
 *
 * <T> is a "type parameter" - it means "any type"
 * When we create a Node, we specify what type T is (e.g., Student, Integer, etc.)
 */
class Node<T> {
    // The data this node carries (the "cargo")
    // T means it can be ANY type: Student, Car, Integer, String, etc.
    T data;

    // Reference to the next node (the "coupler")
    Node<T> next;

    // Constructor: Create a node with given data
    public Node(T data) {
        this.data = data;
        this.next = null;  // Initially not connected to anything
    }
}

What is <T>?

<T> is called a type parameter (or generic type). It's like a placeholder that says "I can hold any type". When you create a node, you specify what T actually is:

  • Node<Student> - A node that holds Student objects
  • Node<Integer> - A node that holds Integer objects
  • Node<Car> - A node that holds Car objects
  • Node<String> - A node that holds String objects

This is the power of generics - one class works for all types!

Step 2: Create the LinkedList Control Class (Generic)

LinkedList Class - Basic Structure (Generic)
/**
 * LinkedList class - Controls and manages a chain of nodes.
 * Responsible for adding, removing, and traversing the list.
 *
 * <T> is the type parameter - this list can hold any type of object!
 */
public class LinkedList<T> {

    // Reference to the first node (we call it "head")
    // This is our entry point to the entire list
    private Node<T> head;

    // Constructor: Create an empty linked list
    public LinkedList() {
        this.head = null;  // Empty list = no head
    }

    // Check if the list is empty
    public boolean isEmpty() {
        return head == null;
    }

    // We will add more methods here...
}

How to Use the Generic LinkedList

When creating a LinkedList, you specify what type it will hold:

// A LinkedList that holds Student objects

// A LinkedList that holds Integer objects
LinkedList<Integer> numberList = new LinkedList<>();

// A LinkedList that holds Car objects

Why "head"?

We call the first node head because:

  • It's the starting point of our list
  • Like the head of a snake - where it begins
  • To access ANY node, we must start from the head and follow the chain

Key Observation: The Same Code Works Every Time!

Did you notice? We used the exact same 3 steps for adding the 1st, 2nd, and 3rd node:

  1. Node newNode = new Node(data) - Create new node
  2. newNode.next = head - Point to current head (even if null!)

This is the power of good algorithm design - no special cases needed for empty vs non-empty list!

Notice: addFirst() Adds in Reverse Order

We called: addFirst(10), then addFirst(20), then addFirst(30)

But the list is: [30 → 20 → 10]

This is because each new node is added at the beginning, pushing older nodes back!

addFirst() Method
/**
 * Adds a new node at the BEGINNING of the list.
 * Time Complexity: O(1) - Always constant time!
 *
 * @param data The object to add (can be any type: Student, Car, Integer, etc.)
 */
public void addFirst(T data) {
    // Step 1: Create a new node
    Node<T> newNode = new Node<>(data);

    // Step 2: New node's next points to current head
    newNode.next = head;

    // Step 3: Head now points to new node
    head = newNode;
}

Step 4: Print the List (Traversal)

Now let's add a method to print all elements - this is how we traverse the list.

printList() Method
/**
 * Prints all elements in the list.
 * Time Complexity: O(n) - Must visit every node
 */
public void printList() {
    // Handle empty list
    if (head == null) {
        System.out.println("List is empty!");
        return;
    }

    // Start from the head
    Node<T> current = head;

    System.out.print("List: ");

    // Traverse until we reach the end (null)
    while (current != null) {
        // Print current node's data
        System.out.print(current.data);

        // Add arrow if not the last node
        if (current.next != null) {
            System.out.print(" -> ");
        }

        // Move to the next node
        current = current.next;
    }

    System.out.println(" -> null");
}

Step 6: Add at End (addLast)

Sometimes we want to add elements at the end of the list.

Algorithm: Add at End

  1. Create a new node with the given data
  2. If the list is empty, make the new node the head
  3. Otherwise, traverse to the last node (where next is null)
  4. Set the last node's next to point to the new node

addLast() Method
/**
 * Adds a new node at the END of the list.
 * Time Complexity: O(n) - Must traverse to the end
 *
 * @param data The object to add (can be any type)
 */
public void addLast(T data) {
    // Step 1: Create a new node
    Node<T> newNode = new Node<>(data);

    // Step 2: If list is empty, new node becomes head
    if (head == null) {
        head = newNode;
        return;
    }

    // Step 3: Traverse to find the last node
    Node<T> current = head;
    while (current.next != null) {
        current = current.next;
    }

    // Step 4: Link the last node to the new node
    current.next = newNode;
}

addAfter() Method
/**
 * Adds a new node AFTER a node containing the target object.
 * Time Complexity: O(n) - Must search for the target
 *
 * @param target The object to search for
 * @param data The object for the new node
 * @return true if successful, false if target not found
 */
public boolean addAfter(T target, T data) {
    // Search for the target node
    Node<T> current = head;

    while (current != null) {
        // Use .equals() to compare objects (not == which compares references)
        if (current.data.equals(target)) {
            // Found it! Create new node
            Node<T> newNode = new Node<>(data);

            // New node points to what target was pointing to
            newNode.next = current.next;

            // Target now points to new node
            current.next = newNode;

            return true;
        }
        current = current.next;
    }

    // Target not found
    System.out.println("Object not found in the list!");
    return false;
}

deleteFirst() Method
/**
 * Deletes the FIRST node from the list.
 * Time Complexity: O(1) - Always constant time!
 *
 * @return The deleted object, or null if list is empty
 */
public T deleteFirst() {
    // Check if list is empty
    if (head == null) {
        System.out.println("List is empty! Nothing to delete.");
        return null;
    }

    // Save the data to return
    T deletedData = head.data;

    // Move head to the next node
    head = head.next;

    return deletedData;
}

Step 9: Delete Last Node (deleteLast)

deleteLast() Method
/**
 * Deletes the LAST node from the list.
 * Time Complexity: O(n) - Must traverse to second-to-last node
 *
 * @return The deleted object, or null if list is empty
 */
public T deleteLast() {
    // Check if list is empty
    if (head == null) {
        System.out.println("List is empty! Nothing to delete.");
        return null;
    }

    // If only one node, delete it
    if (head.next == null) {
        T deletedData = head.data;
        head = null;
        return deletedData;
    }

    // Traverse to find second-to-last node
    Node<T> current = head;
    while (current.next.next != null) {
        current = current.next;
    }

    // Save the last node's data
    T deletedData = current.next.data;

    // Remove the last node
    current.next = null;

    return deletedData;
}

deleteByValue() Method
/**
 * Deletes the first node containing the specified object.
 * Time Complexity: O(n) - Must search for the object
 *
 * @param value The object to delete
 * @return true if deleted, false if not found
 */
public boolean deleteByValue(T value) {
    // Check if list is empty
    if (head == null) {
        System.out.println("List is empty!");
        return false;
    }

    // If head contains the value
    if (head.data.equals(value)) {
        head = head.next;
        return true;
    }

    // Search for the node BEFORE the one to delete
    Node<T> current = head;
    while (current.next != null) {
        if (current.next.data.equals(value)) {
            // Skip over the node to delete
            current.next = current.next.next;
            return true;
        }
        current = current.next;
    }

    // Value not found
    System.out.println("Object not found!");
    return false;
}

Step 11: Search (contains)

contains() Method
/**
 * Checks if the list contains a specific object.
 * Time Complexity: O(n) - Must search through the list
 *
 * @param value The object to search for
 * @return true if found, false otherwise
 */
public boolean contains(T value) {
    Node<T> current = head;

    while (current != null) {
        if (current.data.equals(value)) {
            return true;
        }
        current = current.next;
    }

    return false;
}

Step 12: Get Size

size() Method
/**
 * Returns the number of nodes in the list.
 * Time Complexity: O(n) - Must count all nodes
 *
 * @return The number of nodes
 */
public int size() {
    int count = 0;
    Node<T> current = head;

    while (current != null) {
        count++;
        current = current.next;
    }

    return count;
}

Part 6: Complete Generic LinkedList Implementation

Here is the complete Generic LinkedList class that can store any type of object:

Complete LinkedList.java (Generic)
/**
 * Complete Generic Singly Linked List Implementation
 * CPCS 204 - Data Structures I
 *
 * This linked list can store ANY type of object: Student, Car, Integer, String, etc.
 * The type is specified when creating the list using <T>
 */
public class LinkedList<T> {

    // ==================== Inner Node Class ====================
    /**
     * Node: A generic container that holds data of type T and links to the next node.
     */
    private class Node {
        T data;           // The data stored in this node (any type!)
        Node next;        // Reference to the next node

        Node(T data) {
            this.data = data;
            this.next = null;
        }
    }

    // ==================== LinkedList Properties ====================
    private Node head;  // Reference to the first node

    // ==================== Constructor ====================
    public LinkedList() {
        this.head = null;
    }

    // ==================== Basic Operations ====================

    /** Check if the list is empty */
    public boolean isEmpty() {
        return head == null;
    }

    /** Get the size of the list - O(n) */
    public int size() {
        int count = 0;
        Node current = head;
        while (current != null) {
            count++;
            current = current.next;
        }
        return count;
    }

    // ==================== Insertion Operations ====================

    /** Add at the beginning - O(1) */
    public void addFirst(T data) {
        Node newNode = new Node(data);
        newNode.next = head;
        head = newNode;
    }

    /** Add at the end - O(n) */
    public void addLast(T data) {
        Node newNode = new Node(data);

        if (head == null) {
            head = newNode;
            return;
        }

        Node current = head;
        while (current.next != null) {
            current = current.next;
        }
        current.next = newNode;
    }

    /** Add after a specific object - O(n) */
    public boolean addAfter(T target, T data) {
        Node current = head;

        while (current != null) {
            if (current.data.equals(target)) {
                Node newNode = new Node(data);
                newNode.next = current.next;
                current.next = newNode;
                return true;
            }
            current = current.next;
        }
        return false;
    }

    // ==================== Deletion Operations ====================

    /** Delete the first node - O(1) */
    public T deleteFirst() {
        if (head == null) {
            System.out.println("List is empty!");
            return null;
        }
        T data = head.data;
        head = head.next;
        return data;
    }

    /** Delete the last node - O(n) */
    public T deleteLast() {
        if (head == null) {
            System.out.println("List is empty!");
            return null;
        }

        if (head.next == null) {
            T data = head.data;
            head = null;
            return data;
        }

        Node current = head;
        while (current.next.next != null) {
            current = current.next;
        }
        T data = current.next.data;
        current.next = null;
        return data;
    }

    /** Delete by object - O(n) */
    public boolean deleteByValue(T value) {
        if (head == null) {
            return false;
        }

        if (head.data.equals(value)) {
            head = head.next;
            return true;
        }

        Node current = head;
        while (current.next != null) {
            if (current.next.data.equals(value)) {
                current.next = current.next.next;
                return true;
            }
            current = current.next;
        }
        return false;
    }

    // ==================== Search Operations ====================

    /** Check if object exists - O(n) */
    public boolean contains(T value) {
        Node current = head;
        while (current != null) {
            if (current.data.equals(value)) {
                return true;
            }
            current = current.next;
        }
        return false;
    }

    /** Get object at index - O(n) */
    public T get(int index) {
        if (index < 0) {
            return null;
        }

        Node current = head;
        int count = 0;

        while (current != null) {
            if (count == index) {
                return current.data;
            }
            count++;
            current = current.next;
        }
        return null;  // Index out of bounds
    }

    // ==================== Display ====================

    /** Print the entire list - O(n) */
    public void printList() {
        if (head == null) {
            System.out.println("List is empty!");
            return;
        }

        System.out.print("List: ");
        Node current = head;
        while (current != null) {
            System.out.print(current.data);
            if (current.next != null) {
                System.out.print(" -> ");
            }
            current = current.next;
        }
        System.out.println(" -> null");
    }
}

// Student class (needs toString() and equals() for proper printing and comparison)
class Student {
    String name;
    int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // toString() is called when we print the object
    @Override
    public String toString() {
        return name + "(" + age + ")";
    }

    // equals() is used for comparison in contains(), deleteByValue(), etc.
    @Override
    public boolean equals(Object obj) {
        if (obj instanceof Student) {
            Student other = (Student) obj;
            return this.name.equals(other.name) && this.age == other.age;
        }
        return false;
    }
}

Important: toString() and equals() Methods

When using the LinkedList with custom objects (like Student), make sure to override:

  • toString() - So the object prints nicely
  • equals() - So comparison methods (contains, deleteByValue, addAfter) work correctly

Part 7: Summary

What We Learned

  1. Arrays are fixed because they need contiguous memory for O(1) access
  2. Memory fragmentation can prevent array extension even with free space
  3. OOP concepts (objects with references) can link data together
  4. Linked Lists are a reusable data structure built on this idea
  5. Nodes are generic containers with data of type T and a next reference
  6. Generics (<T>) allow the LinkedList to store ANY type of object
  7. The control class manages all operations (add, remove, search, etc.)

Time Complexity Summary

Operation Time Complexity Notes
addFirst() O(1) Just change head pointer
addLast() O(n) Must traverse to end
addAfter() O(n) Must search for target
deleteFirst() O(1) Just change head pointer
deleteLast() O(n) Must find second-to-last
deleteByValue() O(n) Must search for value
contains() O(n) Must search
get(index) O(n) Must traverse to index
size() O(n) Must count all nodes
printList() O(n) Must visit all nodes
