Introduction

As we previously discussed, in a singly linked list, each node points to the next node, and the last node points to null. Now, we will explore three important variants of linked lists:
  1. Circular Singly Linked List
  2. Doubly Linked List
  3. Circular Doubly Linked List
Each variant solves specific problems and has its own use cases. Let's start with the circular linked list!
Recall: In a singly linked list, each node has:
  • Data - The value stored in the node
  • Next - A reference to the next node (or null if it's the last)

1. Circular Singly Linked List

What is a Circular Linked List?

A Circular Singly Linked List is a variation of the singly linked list where the last node's next pointer points back to the first node instead of null. This creates a circular chain with no end!
Think of it like a circle of friends holding hands. The last person doesn't let go - they hold hands with the first person, completing the circle!

Visual Representation

Key Observation: In a circular linked list, there is no null pointer! The last node always points back to the first node.

Real-World Applications

Application 1: CPU Process Scheduling (Round Robin)

Imagine a CPU needs to execute 3 processes: P1, P2, and P3. Using a circular linked list:

  • CPU gives P1 a time slice (e.g., 5 milliseconds)
  • Then moves to P2 (5 milliseconds)
  • Then moves to P3 (5 milliseconds)
  • Then automatically goes back to P1 and continues the cycle!

This is called Round Robin Scheduling - a circular linked list is perfect for this!

Application 2: Multiplayer Board Games

Think of a game like Monopoly with 4 players:

  • Player 1 takes their turn
  • Then Player 2, Player 3, Player 4
  • Then it goes back to Player 1 automatically!

A circular linked list naturally represents this turn-taking pattern.

Implementation with Head and Tail

To make our implementation easier, we'll use both head and tail pointers. The tail pointer makes adding to the end much more efficient - O(1) instead of O(n)!

Node Class for Circular Linked List

Node.java - Same as Singly Linked List
public class Node {
    int data;
    Node next;

    public Node(int data) {
        this.data = data;
        this.next = null;
    }
}

Adding the First Node

1 Initial State - Empty List

Both head and tail are null

2 Create First Node (data = 10)

When we add the first node:

  • head and tail both point to this node
  • The node's next points to itself!
Adding First Node
// If list is empty (head == null)
Node newNode = new Node(10);
head = newNode;
tail = newNode;
tail.next = head;  // Points to itself - making it circular!

Adding the Second Node (at the end)

3 Add Second Node (data = 20)

Steps to add at the end:

  1. Create new node
  2. New node's next points to head
  3. Current tail's next points to new node
  4. Update tail to new node

Adding the Third Node

4 Add Third Node (data = 30)

Same process - tail now points to node with 30, and 30's next points back to head!

Complete Implementation

CircularLinkedList.java
public class CircularLinkedList {
    Node head;
    Node tail;

    // Add node at the end
    public void addLast(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            // First node - points to itself
            head = newNode;
            tail = newNode;
            tail.next = head;
        } else {
            // Add after tail, update circular reference
            newNode.next = head;     // New node points to head
            tail.next = newNode;     // Old tail points to new node
            tail = newNode;          // Update tail
        }
    }

    // Add node at the beginning
    public void addFirst(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            head = newNode;
            tail = newNode;
            tail.next = head;
        } else {
            newNode.next = head;     // New node points to old head
            head = newNode;          // Update head
            tail.next = head;        // Tail must point to new head!
        }
    }
}

Traversing a Circular Linked List

DANGER: Infinite Loop!

Since there is no null to stop at, using a regular while loop like while (current != null) will run forever!

We need a special condition to stop. We can either:
  1. Use a do-while loop and stop when we come back to head
  2. Use a while loop with a careful condition

Method 1: Using do-while Loop

Traversal with do-while
public void printList() {
    if (head == null) {
        System.out.println("List is empty");
        return;
    }

    Node current = head;

    // do-while ensures we visit at least once, then stop when back at head
    do {
        System.out.print(current.data + " -> ");
        current = current.next;
    } while (current != head);

    System.out.println("(back to head)");
}

// Output: 10 -> 20 -> 30 -> (back to head)
Why do-while?

The do-while loop executes at least once before checking the condition. This is perfect because:

  • We start at head and want to process it
  • We keep going until we return to head
  • If we used while (current != head), we'd never enter the loop!

Method 2: Using while Loop with Counter

Traversal with while Loop (Alternative)
public void printListWhile() {
    if (head == null) {
        System.out.println("List is empty");
        return;
    }

    Node current = head;
    boolean firstIteration = true;

    while (firstIteration || current != head) {
        System.out.print(current.data + " -> ");
        current = current.next;
        firstIteration = false;
    }

    System.out.println("(back to head)");
}
Best Practice: The do-while approach is cleaner and more commonly used for circular list traversal.

2. Doubly Linked List

The Problem with Singly Linked Lists

Professor, what if I'm in the middle of a singly linked list and I need to go back to the previous node?
Excellent question! In a singly linked list, you cannot go backwards. Once you move forward, you've lost the reference to the previous node. To go back, you'd have to start from the head again - that's O(n) every time!
A Doubly Linked List solves this problem by having two pointers in each node:
  • next - Points to the next node (forward direction)
  • prev - Points to the previous node (backward direction)
This allows bidirectional traversal!

Real-World Applications

Application 1: Browser History

When you browse the web:

  • Click "Back" button - go to previous page (use prev pointer)
  • Click "Forward" button - go to next page (use next pointer)

A doubly linked list is perfect for browser navigation!

Node Class for Doubly Linked List

Adding Node at the End - Step by Step

1 Initial State - Empty List

Both head and tail are null

2 Add First Node (data = 10)

When list is empty:

  • Create new node
  • Both head and tail point to it
  • Both prev and next are null
3 Add Second Node (data = 20) at End

Steps:

  1. Create new node with data 20
  2. Set new node's prev to current tail (node 10)
  3. Set current tail's next to new node
  4. Update tail to new node
4 Add Third Node (data = 30) at End

Same process repeats - now we have 3 nodes connected bidirectionally!

Complete Implementation

DoublyLinkedList.java
public class DoublyLinkedList {

    // Inner Node class
    class Node {
        int data;
        Node next;
        Node prev;

        public Node(int data) {
            this.data = data;
            this.next = null;
            this.prev = null;
        }
    }

    Node head;
    Node tail;

    // Add node at the end
    public void addLast(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            // First node - both head and tail point to it
            head = newNode;
            tail = newNode;
        } else {
            // Link new node with current tail
            newNode.prev = tail;     // New node's prev points to old tail
            tail.next = newNode;     // Old tail's next points to new node
            tail = newNode;          // Update tail to new node
        }
    }

    // Add node at the beginning
    public void addFirst(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            head = newNode;
            tail = newNode;
        } else {
            newNode.next = head;     // New node points to old head
            head.prev = newNode;     // Old head's prev points to new node
            head = newNode;          // Update head to new node
        }
    }

    // Print list forward
    public void printForward() {
        Node current = head;
        System.out.print("Forward: ");
        while (current != null) {
            System.out.print(current.data + " <-> ");
            current = current.next;
        }
        System.out.println("null");
    }

    // Print list backward (starting from tail!)
    public void printBackward() {
        Node current = tail;
        System.out.print("Backward: ");
        while (current != null) {
            System.out.print(current.data + " <-> ");
            current = current.prev;
        }
        System.out.println("null");
    }
}
Key Advantage: In a doubly linked list, we can traverse in both directions! This makes operations like deleting a specific node much easier - we don't need to track the previous node separately.

3. Circular Doubly Linked List

A Circular Doubly Linked List combines the features of both:
  • Circular - Last node connects back to first node
  • Doubly - Each node has both next and prev pointers
This creates a bidirectional circle where you can traverse infinitely in both directions!
Think of it as a merry-go-round where you can walk in either direction and you'll eventually return to where you started!

Visual Representation

Key Points:
  • tail.next = head (last node's next points to first)
  • head.prev = tail (first node's prev points to last)
  • No null pointers anywhere!
  • Can traverse infinitely in both directions

Real-World Application

Application: Media Player with Repeat Mode

When "Repeat All" is enabled on a playlist:

  • After the last track, go back to the first track (circular)
  • User can press "Next" or "Previous" (doubly)
  • Never stops - keeps cycling through the playlist!

Implementation

CircularDoublyLinkedList.java
public class CircularDoublyLinkedList {

    // Inner Node class
    class Node {
        int data;
        Node next;
        Node prev;

        public Node(int data) {
            this.data = data;
        }
    }

    Node head;
    Node tail;

    // Add node at the end
    public void addLast(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            // First node - points to itself in both directions
            head = newNode;
            tail = newNode;
            newNode.next = newNode;  // Points to itself
            newNode.prev = newNode;  // Points to itself
        } else {
            // Insert between tail and head
            newNode.prev = tail;     // New node's prev = old tail
            newNode.next = head;     // New node's next = head (circular!)
            tail.next = newNode;     // Old tail's next = new node
            head.prev = newNode;     // Head's prev = new node (circular!)
            tail = newNode;          // Update tail
        }
    }

    // Add node at the beginning
    public void addFirst(int data) {
        Node newNode = new Node(data);

        if (head == null) {
            head = newNode;
            tail = newNode;
            newNode.next = newNode;
            newNode.prev = newNode;
        } else {
            // Insert between tail and head
            newNode.next = head;     // New node's next = old head
            newNode.prev = tail;     // New node's prev = tail (circular!)
            head.prev = newNode;     // Old head's prev = new node
            tail.next = newNode;     // Tail's next = new node (circular!)
            head = newNode;          // Update head
        }
    }

    // Print list (be careful of infinite loop!)
    public void printList() {
        if (head == null) {
            System.out.println("List is empty");
            return;
        }

        Node current = head;
        System.out.print("List: ");

        do {
            System.out.print(current.data + " <-> ");
            current = current.next;
        } while (current != head);

        System.out.println("(back to head)");
    }
}
Remember: Always use do-while with the condition current != head to avoid infinite loops when traversing circular lists!

Comparison of All Types

Feature Singly Linked List Circular Singly Doubly Linked List Circular Doubly
Traversal Direction Forward only Forward only (circular) Forward & Backward Forward & Backward (circular)
Last Node Points To null Head (first node) null Head (first node)
First Node's prev N/A N/A null Tail (last node)
Memory per Node Data + 1 pointer Data + 1 pointer Data + 2 pointers Data + 2 pointers
Go to Previous Node O(n) - start from head O(n) - start from head O(1) - use prev O(1) - use prev
Use Case Simple lists Round-robin, cycling Browser history Media players with repeat

Summary

When to Use Each Type:
  • Singly Linked List: When you only need forward traversal and want minimal memory usage
  • Circular Singly: When you need continuous cycling (CPU scheduling, turn-based games)
  • Doubly Linked List: When you need bidirectional traversal (undo/redo, browser history)
  • Circular Doubly: When you need both circular and bidirectional access (music player with repeat)
