📖 Introduction to Algorithm Analysis

Why Algorithm Analysis?

Algorithm analysis helps us predict performance before running code. Two algorithms solving the same problem can differ by millions of times in speed for large inputs.

  • Time Complexity: How much time an algorithm takes relative to input size n
  • Space Complexity: How much memory an algorithm uses
  • Scalability: How well the algorithm holds up as n grows huge

Teaching Examples
Example 1 Single Loop
☕ Java Code ▼
for (int i = 0; i < n; i++) { sum = sum + i; // 1 operation }
→

Loop runs: n times  ×  1 operation inside = n

f(n) = n
Example 2 Nested Loops (n × n)
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { matrix[i][j] = 0; // 1 operation } }
→

Outer loop: n times  ×  Inner loop: n times  ×  1 op = n²

f(n) = n²
Example 3 Sequential Loops (Add Them)
☕ Java Code▼
for (int i = 0; i < n; i++) { sum += i; // 1 operation } for (int j = 0; j < n*n; j++) { delta += 1; // 1 operation }
1

First loop: runs n times → n ops

2

Second loop: runs n² times → n² ops

+

Sequential → Add: T(n) = n + n²

f(n) = n² + n
Example 4 Logarithmic Loop (Halving)
☕ Java Code▼
while (n > 0) { System.out.println(n); n = n / 2; }
1

After each iteration n is halved: n → n/2 → n/4 → n/8 → … → 1

2

After k iterations: n / 2ᵏ = 1 → 2ᵏ = n → k = log₂(n)

f(n) = log₂(n)
Example 5 Triangular Inner Loop (j < i)
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = 0; j < i; j++) { sum += i * j; // 1 operation } }
1

Inner loop runs: 0, 1, 2, 3, …, n−1 times for each i

2

Total = 0 + 1 + 2 + … + (n−1) = n(n−1)/2

3

Expand: n(n−1)/2 = (n²−n)/2 = n²/2 − n/2

f(n) = (1/2)n² − (1/2)n

② Step 2 — Converting Polynomial to Big O

1
Keep highest-order termThe term with the largest exponent
2
Drop all coefficients76,000,000n² → n²
3
Drop lower-order termsn² + n + 5 → n²
4
Constant → O(1)f(n) = 42 → O(1)
Why? As n → ∞ the highest-order term dominates completely. e.g. for f(n) = 4n² + 3n + 10: when n = 1,000, the 4n² part = 4,000,000 dwarfs 3n (3,000) and 10.

③ Step 3 — Calculating Execution Time

The Formula

If an algorithm is O(g(n)), then its running time is modelled as:

T(n) = c × g(n)

Where c is a hardware/implementation constant. Use a known data point (a given n and time) to find c, then use c to predict any other time.

1
Find cPlug in the given (n, time) pair
2
Use cPlug in the target n to get T(n)
3
Find nPlug in the target time and solve for n
Teaching Examples
Example 1 O(n) — Find time for new n
Algorithm is O(n). For n = 10, time = 2 ms. Find time for n = 500.
1

T(n) = c × n

2

Find c:   2 = c × 10  →  c = 2/10 = 1/5

3

T(500) = (1/5) × 500 = 100 ms

✅ Answer: 100 milliseconds
Example 2 O(n²) — Find time for new n
Algorithm is O(n²). For n = 4, time = 10 ms. Find time for n = 16.
1

T(n) = c × n²

2

10 = c × 4² = c × 16  →  c = 10/16 = 5/8

3

T(16) = (5/8) × 16² = (5/8) × 256 = 160 ms

✅ Answer: 160 milliseconds
Example 3 O(log₂ n) — Find time for new n
Algorithm is O(log₂ n). For n = 16, time = 28 ms. Find time for n = 64.
1

T(n) = c × log₂(n)

2

28 = c × log₂(16) = c × 4  →  c = 7

3

T(64) = 7 × log₂(64) = 7 × 6 = 42 ms

✅ Answer: 42 milliseconds
Example 4 O(n²) — Find input size n
Algorithm is O(n²). For n = 10, time = 30 ms. Find n when time = 120 ms.
1

T(n) = c × n²

2

30 = c × 100  →  c = 3/10

3

120 = (3/10) × n²  →  n² = 400  →  n = 20

✅ Answer: n = 20

🧪 Practice Problems

Part A — Java Code → Polynomial → Big O
Practice 1 Simple Counter Loop
☕ Java Code▼
int count = 0; for (int i = 0; i < n; i++) { count++; // how many operations? }

✅ Solution

1

Loop runs n times, 1 op each → T(n) = n

f(n) = n  O(n)
Practice 2 Nested Loop (j starts at i)
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = i; j < n; j++) { sum += array[i][j]; } }

✅ Solution

1

i=0: inner runs n times  |  i=1: n−1  |  …  |  i=n−1: 1

2

Total = n + (n−1) + … + 1 = n(n+1)/2

3

Expand → n²/2 + n/2

f(n) = n²/2 + n/2  O(n²)
Practice 3 Triple Nested Loop
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { for (int k = 0; k < n; k++) { cube[i][j][k] = i + j + k; } } }

✅ Solution

1

n × n × n = n³

f(n) = n³  O(n³)
Practice 4 Loop Doubles Each Step
☕ Java Code▼
int i = 1; while (i < n) { System.out.println(i); i = i * 2; }

✅ Solution

1

i = 1, 2, 4, 8, …, 2ᵏ   (doubles each time)

2

Stops when 2ᵏ ≥ n → k = log₂(n)

f(n) = log₂(n)  O(log n)
Practice 5 Mixed: Nested + Single Loop
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { matrix[i][j] = 0; } } for (int k = 0; k < n; k++) { sum += array[k]; }

✅ Solution

1

Nested loop: n²

2

Single loop: n

+

Sequential → add: n² + n

f(n) = n² + n  O(n²)
Practice 6 Search with Early Exit
☕ Java Code▼
boolean found = false; for (int i = 0; i < n && !found; i++) { if (array[i] == target) { found = true; } }

✅ Solution

1

Loop runs at most n times (worst case: target not found)

f(n) = n  O(n)
Practice 7 Triangular Star Pattern
☕ Java Code▼
for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { System.out.print("*"); } System.out.println(); }

✅ Solution

1

i=1: inner 1 time  |  i=2: 2  |  …  |  i=n: n

2

Total = 1+2+…+n = n(n+1)/2

f(n) = n²/2 + n/2  O(n²)
Practice 8 Constant Time Operations
☕ Java Code▼
int first = array[0]; int last = array[n - 1]; int mid = array[n / 2]; int sum = first + last + mid;

✅ Solution

1

4 simple statements, no loops — independent of n

f(n) = 4  O(1)
Practice 9 Decreasing Loop
☕ Java Code▼
for (int i = n; i > 0; i--) { sum += array[i - 1]; }

✅ Solution

1

Counts down: n, n−1, …, 1 → exactly n iterations

f(n) = n  O(n)
Practice 10 Step by 2
☕ Java Code▼
for (int i = 0; i < n; i += 2) { process(array[i]); }

✅ Solution

1

i visits: 0, 2, 4, … → n/2 iterations

2

f(n) = n/2 — coefficient dropped in Big O

f(n) = n/2  O(n)

💡 Halving the loop doesn't change the Big O class — constants disappear.

Practice 11 Two Different Variables (n, m)
☕ Java Code▼
for (int i = 0; i < n; i++) { for (int j = 0; j < m; j++) { matrix[i][j] = i + j; } }

✅ Solution

1

Outer: n  |  Inner: m  →  Total: n × m

!

n and m are independent — cannot simplify to n²

f(n,m) = n × m  O(n·m)
Practice 12 Half-Array Nested
☕ Java Code▼
for (int i = 0; i < n/2; i++) { for (int j = 0; j < n/2; j++) { sum += matrix[i][j]; } }

✅ Solution

1

(n/2) × (n/2) = n²/4

2

1/4 is a constant coefficient → dropped

f(n) = n²/4  O(n²)
Practice 13 Reverse Triangle
☕ Java Code▼
for (int i = n; i > 0; i--) { for (int j = 0; j < i; j++) { System.out.print("*"); } }

✅ Solution

1

i=n: n ops  |  i=n−1: n−1  |  …  |  i=1: 1

2

Total = n + (n−1) + … + 1 = n(n+1)/2

f(n) = n²/2 + n/2  O(n²)
Practice 14 Loop Divides by 3
☕ Java Code▼
int i = n; while (i > 1) { System.out.println(i); i = i / 3; }

✅ Solution

1

i = n, n/3, n/9, … After k steps: n/3ᵏ = 1 → k = log₃(n)

f(n) = log₃(n)  O(log n)
Practice 15 Three Sequential Loops
☕ Java Code▼
for (int i = 0; i < n; i++) { array[i] = i; } for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { matrix[i][j] = i * j; } } for (int i = 0; i < n; i++) { sum += array[i]; }

✅ Solution

1

Loop 1: n   |   Loop 2 (nested): n²   |   Loop 3: n

+

Sequential → n + n² + n = n² + 2n

f(n) = n² + 2n  O(n²)
Part B — Polynomial → Big O
Practice 20 Convert 5 Polynomials to Big O

Convert each polynomial to its tightest Big O:

  1. f(n) = 5n³ + 2n² + 100
  2. f(n) = 1000n + 500
  3. f(n) = n² log(n) + 3n²
  4. f(n) = 2ⁿ + n³
  5. f(n) = 15

✅ Solutions

1

5n³ + 2n² + 100 → highest term 5n³, drop coeff →  O(n³)

2

1000n + 500 → highest term 1000n, drop coeff →  O(n)

3

n² log n + 3n² → n² log n dominates n² →  O(n² log n)

4

2ⁿ + n³ → exponential beats polynomial →  O(2ⁿ)

5

f(n) = 15 → constant, no n →  O(1)

Algorithm is O(n). For n = 50, time = 5 ms. Find time when n = 200.

5 = c × 50 → c = 1/10

3

T(200) = (1/10) × 200 = 20 ms

✅ Solution

1

T(n) = c × n²

2

25 = c × 25 → c = 1

3

T(20) = 1 × 400 = 400 ms

Practice 18 O(log₂ n) — Find Time
Algorithm is O(log₂ n). For n = 32, time = 10 ms. Find time when n = 256.

✅ Solution

T(n) = c × log₂(n)

2

10 = c × log₂(32) = c × 5 → c = 2

3

T(256) = 2 × log₂(256) = 2 × 8 = 16 ms

🔥 Challenging Problems

Challenge 1 Logarithmic Outer × Linear Inner
☕ Java Code▼
for (int i = 1; i < n; i = i * 2) { for (int j = 0; j < n; j++) { process(i, j); } }

✅ Solution

1

Outer loop: i doubles → runs log₂(n) times

2

Inner loop: runs n times each outer iteration

×

Total = n × log₂(n)

f(n) = n·log₂(n)  O(n log n)
Challenge 2 Tricky — j is Never Reset
☕ Java Code▼
int i = 0, j = 0; while (i < n) { while (j < n && array[i][j] == 1) { j++; } i++; }

✅ Solution — careful, this looks like O(n²) but isn't!

!

j is NOT reset when i increments — it keeps its value between outer iterations

1

i increments n times total

2

j increments at most n times total (never goes backwards)

3

Maximum total operations = 2n

f(n) = 2n  O(n)
Challenge 3 Recursive Method
☕ Java Code▼
void mystery(int n) { if (n <= 1) return; for (int i = 0; i < n; i++) { System.out.println(i); } mystery(n / 2); mystery(n / 2); }

✅ Solution

1

Recurrence: T(n) = 2T(n/2) + n

2

Each level does n total work, depth of recursion = log₂(n)

3

Total = n × log₂(n) (this is Merge Sort's pattern!)

O(n log n)
Challenge 4 Square Root Outer Loop
☕ Java Code▼
for (int i = 1; i * i <= n; i++) { for (int j = 0; j < n; j++) { process(i, j); } }

✅ Solution

1

Outer: runs while i² ≤ n → i ≤ √n → √n iterations

2

Inner: n iterations each → Total = √n × n = n^(3/2)

f(n) = n^(3/2)  O(n√n)
Challenge 5 Triple Triangular
☕ Java Code▼
for (int i = 1; i <= n; i++) { for (int j = 1; j <= i; j++) { for (int k = 1; k <= j; k++) { sum++; } } }

✅ Solution

1

For each i, inner two loops give triangular(i) = i(i+1)/2

2

Total = Σᵢ₌₁ⁿ i(i+1)/2 = n(n+1)(n+2)/6 ≈ n³/6

f(n) ≈ n³/6  O(n³)
Challenge 6 Compare Two Algorithms
Algorithm A: O(n log n)  |  n = 8 → 24 ms
Algorithm B: O(n²)  |  n = 10 → 50 ms
Which is faster for n = 100?

✅ Solution

A

24 = c × 8 × log₂(8) = c × 24 → c = 1
T(100) = 1 × 100 × log₂(100) ≈ 100 × 6.64 = 664 ms

B

50 = c × 100 → c = 0.5
T(100) = 0.5 × 10,000 = 5,000 ms

🏆 Algorithm A wins! (664 ms vs 5,000 ms)
