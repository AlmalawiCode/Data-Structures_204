Why Do We Traverse a Graph?

The fundamental operations that make graphs useful

A graph stores connections — but to be useful we need to visit all reachable nodes systematically. Traversal is the foundation of almost every graph algorithm:

1🔍

Searching

Find if node B is reachable from A — like checking if two computers are on the same network.

───▶
2🗺️

Pathfinding

Find the shortest or any path between two nodes — the core of GPS navigation and routing.

───▶
3🔗

Connectivity

Find all nodes in a connected component — used in social networks, disease spread models.

Two strategies, one goal Both DFS and BFS visit every reachable vertex exactly once. The difference is order: DFS dives deep first, BFS explores level by level. Use the simulator below to see exactly how each one works.

Professional Implementation

How DFS and BFS are actually written in code — Java (standard implementation)

Key rule in both algorithms: Mark visited[node] = true as soon as you arrive at a node. This prevents any other path from revisiting the same node. 
// DFS — explore one neighbor at a time, backtrack when stuck
// graph: adjacency list as List<List<Integer>>

import java.util.*;

public static List<Integer> dfs(List<List<Integer>> graph, int start) {
    boolean[] visited = new boolean[graph.size()];
    List<Integer>  result = new ArrayList<>();
    Stack<Integer> stack  = new Stack<>();

    visited[start] = true;           // mark when visiting
    stack.push(start);
    result.add(start);               // record first visit

    while (!stack.isEmpty()) {
        int node = stack.peek();       // look at top — do NOT pop yet
        int next = -1;

        for (int neighbor : graph.get(node)) {
            if (!visited[neighbor]) { next = neighbor; break; } // pick ONE
        }

        if (next != -1) {
            visited[next] = true;     // mark when visiting
            stack.push(next);          // suspend current, go deeper
            result.add(next);
        } else {
            stack.pop();               // no unvisited neighbors → backtrack
        }
    }
    return result;
}
How this works stack.peek() looks at the current node without removing it — the node stays suspended on the stack while you go deeper. When you find an unvisited neighbor, you push it (go deeper). When none remain, you stack.pop() (backtrack). The stack always holds your current path from start to current node. 
// BFS using a Queue (LinkedList for O(1) poll)
// graph: adjacency list as List<List<Integer>>

import java.util.*;

public static List<Integer> bfs(List<List<Integer>> graph, int start) {
    boolean[] visited = new boolean[graph.size()]; // visited[] boolean array
    List<Integer>   result = new ArrayList<>();
    Queue<Integer>  queue  = new LinkedList<>();

    visited[start] = true;               // ← mark BEFORE enqueuing
    queue.add(start);

    while (!queue.isEmpty()) {
        int node = queue.poll();           // dequeue from front
        result.add(node);                  // explore (output) this node

        for (int neighbor : graph.get(node)) {
            if (!visited[neighbor]) {
                visited[neighbor] = true; // ← mark BEFORE enqueuing
                queue.add(neighbor);
            }
        }
    }
    return result;
}
Identical pattern to DFS — the only difference is the data structure: Stack with stack.pop() (LIFO) vs Queue with queue.poll() (FIFO). The visited[] marking rule is exactly the same in both.
