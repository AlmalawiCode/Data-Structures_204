Binary Search Tree (BST) - Comprehensive Question

Total Marks: 15
Question: You are given the following keys to insert into an initially empty Binary Search Tree (BST). Insert them in the given order from left to right.
Keys: 50, 25, 75, 10, 30, 60, 90, 5, 15, 28, 55, 85
Insertion Rule: For each key k, if k < current node, go left; if k > current node, go right.
Part A 3 Marks

Construct the Binary Search Tree by inserting the keys in the given order. Draw the final tree.

Part B 3 Marks

Write the Post-Order traversal of the tree you constructed.

Post-Order: Left subtree → Right subtree → Root
Post-Order: ________________________________________________________
Part C 3 Marks

True or False: Answer the following questions about the tree.

Statement True False
1. This BST is a Complete Binary Tree
2. This BST is a Full Binary Tree
3. The height of this BST is 4
Part D 3 Marks

Delete node 25 from the tree. Draw the resulting tree after deletion.

Part E 3 Marks

Write a recursive method that counts the number of nodes containing even numbers in a BST.

Method signature: int countEven(Node root)
public int countEven(Node root) { }

Solution

Part A: BST Construction (3 Marks)

Insertion Order: 50, 25, 75, 10, 30, 60, 90, 5, 15, 28, 55, 85

Tree Properties:

  • Total nodes: 12
  • Height: 3 (counting edges) or 4 (counting levels)
  • Even numbers: 50, 10, 30, 60, 90, 28 (6 nodes)
  • Odd numbers: 25, 75, 5, 15, 55, 85 (6 nodes)

Part B: Post-Order Traversal (3 Marks)

Post-Order (Left → Right → Root):

5, 15, 10, 28, 30, 25, 55, 60, 85, 90, 75, 50

Step-by-step:

  1. Process left subtree of 50 → left subtree of 25 → left subtree of 10: 5
  2. Right subtree of 10: 15
  3. Root 10: 10
  4. Right subtree of 25 → left subtree of 30: 28
  5. Root 30: 30
  6. Root 25: 25
  7. Right subtree of 50 → left subtree of 75 → left subtree of 60: 55
  8. Root 60: 60
  9. Right subtree of 75 → left subtree of 90: 85
  10. Root 90: 90
  11. Root 75: 75
  12. Root 50: 50

Part C: True or False (3 Marks)

Statement Answer Explanation
1. This BST is a Complete Binary Tree FALSE Level 3 is not filled left-to-right (node 30 has no right child)
2. This BST is a Full Binary Tree FALSE Nodes 30, 60, 90 have only one child each
3. The height of this BST is 4 TRUE 4 levels (0→1→2→3) or height = 3 if counting edges
Definitions:
• Complete Binary Tree: All levels completely filled except possibly the last, which is filled from left to right.
• Full Binary Tree: Every node has either 0 or 2 children (no node has exactly 1 child).

Part D: Delete Node 25 (3 Marks)

Note: Both methods are correct. Students may use either one.

Method 1: In-order Successor

Steps:

  1. Node 25 has two children (10 and 30)
  2. Find in-order successor: smallest value in right subtree = 28
  3. Replace 25 with 28
  4. Delete original node 28

Method 2: In-order Predecessor

Steps:

  1. Node 25 has two children (10 and 30)
  2. Find in-order predecessor: largest value in left subtree = 15
  3. Replace 25 with 15
  4. Delete original node 15

Part E: Count Even Nodes (3 Marks)

Recursive Solution:

public int countEven(Node root) { // Base case: empty tree if (root == null) { return 0; } // Check if current node is even int count = 0; if (root.data % 2 == 0) { count = 1; } // Recursively count in left and right subtrees return count + countEven(root.left) + countEven(root.right); }

Alternative (Compact Version):

public int countEven(Node root) { if (root == null) return 0; int isEven = (root.data % 2 == 0) ? 1 : 0; return isEven + countEven(root.left) + countEven(root.right); }

For this tree: countEven returns 6 (nodes: 50, 10, 30, 60, 90, 28)
