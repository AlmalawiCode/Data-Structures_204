🔑 Why Do We Need Heap?

Sometimes we need to always access the most important element quickly — not based on when it arrived, but based on its priority. The heap is the perfect data structure for this. It gives us O(log n) for both insertion and deletion of the highest or lowest priority element.

By the end of this tutorial you will understand: what a heap is, how to build one step by step, how to insert and delete efficiently using percolate up and percolate down, and most importantly — why the heap is far better than sorting an array every time.

2What Does "Heap" Mean?

📚 The Literal English Meaning

Before we study the data structure, let us understand the English word itself. In English, a heap means a pile or collection of things placed together — not perfectly arranged, but loosely organized. For example:

📚

A heap of books

Books piled on top of each other. The top book is easy to reach, but the pile is not neatly sorted.

🧱

A heap of sand

Sand piled high. The top is the highest point, but the individual grains are not in any order.

🎃

Heap Data Structure

Data arranged so the "most important" item is always at the top — like the biggest item sitting at the peak of the pile!

Key Insight: Just like a physical heap where the largest or most important item naturally sits at the top and is easiest to reach, a heap data structure always keeps the most important element (maximum or minimum) at the root for instant O(1) access.

💻 Heap in Computer Science

In computer science, a heap is a special tree-based data structure with these characteristics:

  • The data is arranged in order (every parent satisfies a rule with respect to its children)
  • But the data is NOT fully sorted (we will see this clearly later!)
  • The root is always the maximum or minimum element
  • It must be stored as a complete binary tree (we will explain exactly why this matters)
⚠ Important Note: Do NOT confuse the "heap" data structure with the "heap memory" area in operating systems. These are two completely different things that happen to share the same name!

3Why Do We Need Heap? — The Hospital Emergency

Imagine you are asked to build a software system for a hospital emergency room. Patients arrive one by one and each is given a triage score (a number from 1 to 100). The higher the score, the more critical and urgent the patient's condition is.

☾︎ What Is a Triage Score? In hospital emergency rooms, triage is the process of assessing patients and prioritizing them by the urgency of their condition. A high triage score means the patient needs immediate treatment. We do NOT treat patients in the order they arrived — we always treat the most critical patient first!

⛔ Approach 1: Sorted Array — The Naive Solution

One student might say: "Doctor, let us just keep all patients in an array sorted in descending order. After each new arrival, sort the array again!"

What happens each time:

  1. Keep all patients in an array
  2. New patient arrives → add to end
  3. Sort the entire array again (descending)
  4. Treat patient at index 0 (highest score)
  5. Remove index 0 → shift all elements left

Yes, this works. But look at the cost!

❌ The Problem with this approach:
  • Every time a new patient arrives, we must sort the entire array
  • The cost of each sort is O(n log n)
  • When we treat the top patient and remove them, we must shift all elements left → another O(n) operation
  • For 7 patients this is fine — but the heap is used beyond hospitals! ✔ The Heap Solution:
  • Insert a new patient: O(log n)
  • Treat and remove the most critical patient: O(log n)
  • The most critical patient is always at the root for instant O(1) access
  • For a dataset with 1 billion records (such as document similarity scores or product rankings) — at most 30 comparisons per insert or delete!

4What Is a Heap?

A heap has two required properties. Both must be satisfied at all times:

🌟 Property 1: Complete Binary Tree

A heap must be stored as a complete binary tree:

  • All levels are completely filled, except possibly the last
  • The last level is filled left to right with no gaps

🌟 Property 2: Heap Property

Every parent node follows an ordering rule:

  • Max-Heap: Parent ≥ both children
  • Min-Heap: Parent ≤ both children
  • This applies to every parent-child pair in the entire tree

🔑 Definition

A Heap is a complete binary tree where every parent node is either greater than or equal to (max-heap) or less than or equal to (min-heap) both of its children. Because of this, the root always holds the maximum (or minimum) element.

6Why Must the Heap Be a Complete Tree?

A student in the class asked: "Doctor, why must the heap be a complete binary tree? Can it not be any binary tree shape?"

Excellent question! The answer is: Memory efficiency and the ability to represent the tree as a simple array.

🔧 The Array Representation Trick

The most powerful feature of the heap is that we can store a complete binary tree perfectly inside a simple array — with no wasted space and no pointers. This works because of a beautiful mathematical property of complete trees.

Using a 0-indexed array (index starts at 0), for any node at index i:

Parent(i)      = ⌊(i − 1) / 2⌋     // integer division (floor), only valid when i > 0
LeftChild(i)  = 2 × i + 1
RightChild(i) = 2 × i + 2

✓ Complete Tree — Perfect Fit, Zero Waste

A complete binary tree with 7 nodes maps perfectly to an array of size 7. Every array position is used. No waste at all!

8Heap Insertion & Percolate Up

📋 The Insertion Algorithm

When we insert a new element into a max-heap:

  1. Step 1 — Add at end: Place the new element at the next available position (end of the array). This maintains the complete tree shape.
  2. Step 2 — Compare with parent: Find the parent using the formula ⌊(i−1)/2⌋. Compare the new element with its parent.
  3. Step 3 — Swap if needed: If the new element is greater than its parent (max-heap), swap them.
  4. Step 4 — Repeat: Move up and continue comparing until we reach the root or the heap property is satisfied.

⬆ Percolate Up (also called "Bubble Up" or "Sift Up")

The new element "percolates up" through the tree — comparing with each parent and rising until it reaches its correct position where the heap property is satisfied. Like a bubble of air rising through water to its natural level.

🎮 Simulation A — Visual Tree Construction

Watch the max-heap grow as a visual binary tree. The key message to understand: we always insert at the next available position, filling left to right, level by level. This is completely different from a BST — we do NOT decide left or right based on the value!

⚠ Heap vs BST: In a Binary Search Tree, smaller values go LEFT and larger values go RIGHT. In a Heap, position is determined by level order (to keep the tree complete), and then value order is enforced by percolate up.

📋 Pseudocode

INSERT(heap, value):
  heap[heap.size] ← value              // add at end (maintain complete tree)
  i ← heap.size
  heap.size ← heap.size + 1
  PERCOLATE_UP(heap, i):
    while i > 0:
      parent ← ⌊(i − 1) / 2⌋
      if heap[i] > heap[parent]:         // max-heap: larger value should be higher
        SWAP(heap[i], heap[parent])
        i ← parent
      else:
        break                           // heap property satisfied — stop

9Heap Deletion & Percolate Down

📋 The Deletion Algorithm

In a heap we always delete the root (the maximum in a max-heap). We never delete from the middle. Here are the steps:

  1. Step 1 — Save and remove root: The root is always the maximum (in a max-heap). Save its value and remove it.
  2. Step 2 — Move last to root: Take the very last element in the array (bottom-right leaf) and place it at the root [0]. Decrease size by 1.
  3. Step 3 — Compare with children: The new root likely violates the heap property. Compare it with its two children.
  4. Step 4 — Swap with larger child: Swap with the larger child (for max-heap), if the root is smaller than it.
  5. Step 5 — Repeat: Continue moving down until the heap property is restored. This is Percolate Down.

⬇ Percolate Down (also called "Sift Down")

After deletion, the element placed at the root "percolates down" through the tree — comparing with its children and sinking until it reaches its correct position. It always swaps with the larger child (max-heap) to maintain the heap property.

📋 Pseudocode

EXTRACT_MAX(heap):
  max ← heap[0]                       // save root (the maximum)
  heap[0] ← heap[heap.size - 1]       // move last element to root
  heap.size ← heap.size - 1
  PERCOLATE_DOWN(heap, 0):
    i ← 0
    while 2×i + 1 < heap.size:           // while node i has at least one child
      left ← 2×i + 1
      right ← 2×i + 2
      larger ← left                     // assume left is the larger child
      if right < heap.size and heap[right] > heap[left]:
        larger ← right
      if heap[i] < heap[larger]:          // heap property violated
        SWAP(heap[i], heap[larger])
        i ← larger
      else:
        break                             // heap property restored — stop
  return max

10Ordered but NOT Sorted — Very Important!

This is one of the most important concepts in this tutorial. You must clearly understand the difference.

📖 The Key Distinction

A heap is ORDERED: every parent satisfies the heap property with respect to its children (parent ≥ children in a max-heap). But a heap is NOT SORTED: the elements are not arranged in descending (or ascending) order throughout the entire tree.

Look carefully at our max-heap: array = [—, 90, 35, 60, 20, 30, 45, 50]

11Complexity Analysis — Why Is Heap So Fast?

📊 Height of the Heap = O(log n)

This is the key insight behind all heap efficiency. Because the heap is a complete binary tree, its height is always:

Height(n) = ⌊log₂(n)⌋
// where n is the number of elements in the heap

// Examples:
n = 7        → height = ⌊log₂(7)⌋   = ⌊2.81⌋  = 2    // at most 2 comparisons going up/down
n = 15      → height = ⌊log₂(15)⌋  = ⌊3.91⌋  = 3
n = 1,000  → height = ⌊log₂(1000)⌋  = ⌊9.97⌋  = 9
n = 1,000,000 → height = ⌊log₂(10⁶)⌋   = ⌊19.93⌋ = 19
n = 1,000,000,000 → height ≈ 29
💡 Think about this! The hospital example has a small number of patients — that is realistic. But the heap is used in many systems that handle truly massive data: search engines ranking document similarity scores, recommendation systems scoring millions of products, GPS navigation finding shortest paths. For one billion records in any such system, inserting or removing the top-priority element requires at most 29 comparisons. Compare that to sorting or shifting a billion elements!

📖 Final Answer: Why Not Just Sort the Array?

Sorting the array works perfectly for a static dataset. But in a dynamic system (patients constantly arriving and leaving, jobs constantly submitted and completed), we need repeated, efficient insert and delete operations. The heap gives us both for O(log n) each — making it ideal for any priority queue application.

12Summary — Everything About the Heap

🏆 The Heap Is a Priority Queue

A heap is the most efficient implementation of a priority queue: a data structure where every element has a priority and the element with the highest (or lowest) priority is always served first. Our hospital emergency room is the perfect real-world model!

Any problem needing the min/max of a dynamic set uses a heap!

🎓 What You Must Remember:
  • A heap is a complete binary tree + heap property (parent ≥ children for max-heap)
  • It is ordered (heap property holds) but NOT sorted
  • Stored as a 0-indexed array: parent = ⌊(i−1)/2⌋, left = 2i+1, right = 2i+2
  • The complete tree constraint ensures zero memory waste and valid index formulas
  • Insert: add at end → percolate up → O(log n)
  • Extract Max: remove root → move last to root → percolate down → O(log n)
  • Far better than sorting an array each time: O(log n) vs O(n log n) per operation
